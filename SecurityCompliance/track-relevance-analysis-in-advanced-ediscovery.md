# <a name="track-relevance-analysis-in-office-365-advanced-ediscovery"></a>在 Office 365 進階電子文件探索中追蹤相關性分析

> [!NOTE]
> 進階電子文件探索需要具有進階合規性附加元件的 Office 365 E3，或適用於您組織的 E5 訂閱。如果您沒有該方案，且想要嘗試進階電子文件探索，您可以[註冊 Office 365 企業版 E5 試用版](https://go.microsoft.com/fwlink/p/?LinkID=698279)。 
  
進階 eDiscovery 中 [相關性追蹤] 索引標籤會顯示在 [標籤] 索引標籤中執行的相關性訓練的計算的有效性，並指出下一個步驟來反覆訓練程序中所採取的相關性。 
  
## <a name="tracking-relevance-training-status"></a>追蹤相關性訓練狀態

1. 檢視詳細資料如下相關性追蹤記錄中的大小寫的問題，以下**問題名稱**] 對話方塊的下列範例所示。 
    
  - **評估**： 此進度列指示器顯示何種程度相關性訓練的執行到此點已達到評估目標方面的錯誤限度。 相關性訓練結果的豐富性也會顯示。 
    
  - **訓練**： 這個可識別的進度指標和工具提示顯示指出相關性訓練結果所展現的穩定性及顯示相關性訓練樣本數字刻度標記的每一項問題。 專家監視反覆相關性訓練程序的進度。 
    
  - **批次計算**： 此進度列指示器提供完成的批次計算的相關資訊。
    
  - **下一步**： 會顯示要執行的下一個步驟的建議。 
    
    在範例中，會顯示此問題： 成功完成的評估，以完成的彩色進度列指示器和核取記號表示。 標記正在進行中，但這種情況仍然會被視為不穩定 （也顯示工具提示中所展現的穩定性狀態）。 下一步步驟建議 「 訓練。 」 
    
    ![相關性追蹤訓練步驟 1](media/a00fe607-680a-48eb-9d61-4565319f7ab6.png)
  
    展開的檢視會顯示其他資訊和選項。 顯示目前錯誤邊界是在評估給定的現有 （已標記） 的評定檔案的目前狀態重新叫用的錯誤邊界。
    
    > [!NOTE]
    >  評估階段可以略過清除每個問題，然後針對 「 所有問題 」 的**評估**核取方塊。 不過，因此，會有這個問題沒有統計資料。 清除 [**評估**] 核取方塊的 > 才可執行評估之前。 在多個問題存在於案例中，核取方塊已清除的每一項問題時，才將略過評估 
  
    當評估就不會完成與第一個範例設定檔的、 評估可能會標示多個檔案的下一步。 
    
    在 [**相關性** \> **追蹤**訓練進度列指示器，工具提示指出到達所展現的穩定性所需的額外樣本估計的數。 這項預估值需要其他訓練提供的指導方針。
    
    ![相關性追蹤訓練](media/98dbc3f5-5238-4d73-9f88-1aa4d77ea729.png)
  
2. 當您準備完成標記和若要繼續訓練時，按一下 [**訓練**。 從載入的檔案設定的其他訓練產生另一個範例一組檔案。 您然後都會傳回到標記並訓練多個檔案的 [標籤] 索引標籤。
    
### <a name="reaching-stable-training-levels"></a>達到穩定訓練層級

評估檔案已得到穩定層級之後，進階電子文件是訓練的準備好進行批次計算。
  
> [!NOTE]
> 通常，三個穩定訓練範例之後下, 一步就是 「 批次計算 」。 可能有例外狀況，例如，若要從先前的範例或植入的檔案時所新增的檔案標記發生變更時。 
  
### <a name="performing-batch-calculation"></a>執行批次計算

批次計算執行為下一個步驟之後訓練成功完成 （時的穩定訓練狀態會顯示進度列中，核取記號和工具提示中的穩定狀態。）批次計算會套用至整個檔案母群體，若要評估的檔案相關性，並將指定相關性分數的相關性訓練期間取得的知識。
  
多個問題時，批次計算是完成每個問題。 批次計算期間處理的所有檔案時的監視進度。 
  
在這裡，建議的下一個步驟為"None"，這表示沒有其他反覆相關性訓練需要這個時候。 下一個階段是**相關性\>決定**] 索引標籤。 
  
如果您想要匯入批次計算後的新檔案，系統管理員可以新增新的負載匯入的檔案。
  
> [!NOTE]
> 如果您按一下 [**取消**] 批次計算期間，處理程序會儲存什麼已經被執行。 如果您執行一次批次計算時，會從最後一個執行點繼續程序。 
  
### <a name="assessing-tagging-consistency"></a>評估標記一致性

如果中已有不一致標記檔案，將會影響分析。 結果不是最佳化或一致性是有疑問時，可以使用進階電子文件探索標記一致性處理程序。 會傳回可能不一致標記檔案的清單，而且您可以檢閱並重新標記，視。
  
> [!NOTE]
> 七個或多個下列評估、 標記一致性可檢視在**相關性**訓練趨之後\>**追蹤** \> **問題** \> **詳細的結果** \> **訓練進度**。 此檢閱一次完成一個問題。 
  
1. 在**相關性\>追蹤**，依序展開 [問題] 列。
    
2. 至右邊的**下一個步驟**中，按一下 [**修改**]。
    
3. [**下一步**] 選項後七個訓練範例為選取**標記不一致**，然後按一下 [**確定]**。
    
4. 選取 [**標記不一致**。 [**標籤**] 索引標籤會開啟顯示來重新標記 [必要時的不一致的清單。 
    
5. 按一下 [**計算**]，將變更提交]。 之後標記不一致的情況下一步 「 訓練。 」

### <a name="keywords"></a>關鍵字

關鍵字是唯一的字串、 單字、 片語或序列進階電子文件所識別檔案是否相關的重要指標為檔案中的字數。 「 包含 」 資料行清單關鍵字和標示為相關，檔案中的權數，「 排除 」 欄會列出關鍵字和標示為不可檔案中的加權相關。
  
進階電子文件會指派誤判或誤判關鍵字加權值。 較高的權數，高到關鍵字上方會出現檔案在批次計算期間指派較高的相關性分數的可能性。 
  
關鍵字進階電子文件清單可用來補充內建專家或間接例行性檢查清單檔案中的任何一點檢閱程序。
  
### <a name="training-progress"></a>訓練進度

**訓練進度**窗格包含訓練進度圖形和品質指標顯示畫面中，如以下範例所示。 
  
![相關性追蹤訓練進度](media/8a5089f5-a162-4246-ae09-bc1921859860.png)
  
 **訓練品質指標**： 顯示標記一致性的評等，如下所示：
  
- **良好**： 一致標記檔案。 （顯示綠色亮）
    
- **中型**： 某些檔案可能不一致標記。 （黃色燈號顯示）
    
- **警告**： 許多檔案可能不一致標記。 （紅色亮顯示）
    
 **訓練進度 graph**: F 量值與相關性訓練週期的天數之後顯示相關性訓練穩定性的程度。 當我們從左邊向右移動整個圖表信賴區間以縮小，由，F-量值，以及進階電子文件至判斷所展現的穩定性的相關性相關性訓練結果時已進行最佳化。
  
> [!NOTE]
> 相關性使用 f2 鍵，其中重新叫用收到精確度為倍權數 F 度量評量。 針對案例具有高豐富性 （超過 25%) 相關性使用 F1 （1:1 對比率）。 **相關性設定**可以設定 F 量值的比例\>**進階的設定**。 
  
### <a name="batch-calculation-results"></a>批次計算結果

**批次計算結果**] 窗格包含已，如下所示的相關性，計分的檔案數目： 
  
- **Success**
    
- **空白**： 不包含文字，例如，僅空格/索引標籤

### <a name="training-statistics"></a>訓練統計資料

**訓練統計資料**] 窗格會顯示的統計資料和進階電子文件相關性訓練結果為基礎的圖形。

- **參數**： 累計計算集到整個案例檔案母體 relation 檢閱相關的統計資料。
