# <a name="set-up-office-365-atp-anti-phishing-and-anti-phishing-policies"></a>設定 Office 365 ATP 防網路釣魚功能及防網路釣魚原則

[ATP 反網路釣魚防護](atp-anti-phishing.md), 部分的[Office 365 高級威脅防護](office-365-atp.md), 可協助保護您的組織免受惡意模擬型網路釣魚攻擊和其他網路釣魚攻擊。 如果您是 Office 365 企業版或安全性系統管理員, 則可以設定 ATP 反網路釣魚原則。 

網路釣魚攻擊受到各種形式的 spear 網路釣魚攻擊或 whaling。 隨著複雜性日益增加, 甚至很難識別某些複雜攻擊的訓練。 幸運的是, Office 365 的高級威脅防護可能會有所説明。 您可以設定 ATP 反網路釣魚原則, 以協助確保您的組織受到這類攻擊的保護。
  
> [!NOTE]
> ATP 反網路釣魚僅適用于高級威脅防護 (ATP)。 ATP 包含在訂閱中, 例如[microsoft 365 企業版](https://www.microsoft.com/microsoft-365/enterprise/home)、 [Microsoft 365 商務](https://www.microsoft.com/microsoft-365/business)版、Office 365 企業版 E5、Office 365 教育版 A5 等等。如果您的組織有不含 Office 365 ATP 的 Office 365 訂閱, 您可能會將 ATP 作為附加元件購買。 如需詳細資訊, 請參閱[office 365 高級威脅防護方案和價格](https://products.office.com/exchange/advance-threat-protection), 以及[Office 365 的高級威脅防護服務說明](https://docs.microsoft.com/office365/servicedescriptions/office-365-advanced-threat-protection-service-description)。 請確定您的組織在 Windows 上使用最新版的 Office 365 專業增強版, 以充分利用 ATP 反網路釣魚保護。 

您也可以使用 Office 365 Exchange Online Protection 的反網路釣魚原則, 其中包含一組有限的反欺騙防護, 其目的是防範驗證型和以 deception 為基礎的攻擊。
  
待辦事項:
  
1. 檢查必要條件。

2. 瞭解您的反網路釣魚和 ATP 反網路釣魚原則選項。

3. 設定反網路釣魚原則或 ATP 反網路釣魚原則。

>[!IMPORTANT]
> 若要瞭解如何套用多種技術, 請造訪[如何結合原則和保護](https://docs.microsoft.com/office365/securitycompliance/how-policies-and-protections-are-combined)。

## <a name="review-the-prerequisites"></a>審查必要條件

- 若要定義 (或編輯) ATP 原則, 您必須獲指派適當的角色。 下表說明一些範例:

    |角色  |指派的位置/方式  |
    |---------|---------|
    |Office 365 全域系統管理員 |註冊購買 Office 365 的人員預設為全域系統管理員。 (請參閱[關於 Office 365 系統管理員角色](https://docs.microsoft.com/office365/admin/add-users/about-admin-roles)以深入瞭解。)         |
    |安全性系統管理員 |Azure Active Directory 系統管理中心 ([https://aad.portal.azure.com](https://aad.portal.azure.com))|
    |Exchange Online 組織管理 |Exchange 系統管理中心 ([https://outlook.office365.com/ecp](https://outlook.office365.com/ecp)) <br>或 <br>  PowerShell Cmdlet (請參閱[Exchange Online PowerShell](https://docs.microsoft.com/powershell/exchange/exchange-online/exchange-online-powershell?view=exchange-ps)) |

    若要深入瞭解角色和許可權, 請參閱[Office 365 安全性&amp;與合規性中心的許可權](permissions-in-the-security-and-compliance-center.md)。

- 您可能會為您的組織設定多個反網路釣魚原則。 Office 365 會依其在 [安全&amp;規範中心] 的 [**反網路釣魚] 頁面**和 [ **ATP 反網路釣魚**] 頁面上列出的順序來強制執行這些原則。 一旦您檢查[原則選項](#learn-about-atp-anti-phishing-policy-options), 請花一些時間來決定您需要的原則數目及每個原則的優先順序。 

- 規劃花大約5-15 分鐘來設定第一次的反網路釣魚原則。

- 最多可允許30分鐘, 以將新的或更新的原則傳播至所有 Office 365 資料中心。

## <a name="set-up-an-anti-phishing-or-atp-anti-phishing-policy"></a>設定反網路釣魚或 ATP 反網路釣魚原則

Office 365 中的每個組織都有一個套用至所有使用者的預設反網路釣魚原則。 您可以建立多個自訂的反網路釣魚原則, 以限定您組織中的特定使用者、群組或網域。 您建立的自訂原則會優先于預設原則。 您可以在 Office 365 安全性&amp;與合規性中心內新增、編輯和刪除反網路釣魚原則。
  
1. 移至[https://protection.office.com](https://protection.office.com) , 並使用您的公司或學校帳戶登入。 

2. 在 Office 365 安全性&amp;合規性中心的左功能窗格中, 選擇 [**威脅管理**] 底下的 [**原則**]。

3. 在 [**原則**] 頁面上, 選擇 [**反網路釣魚**] 或 [ **ATP 反網路釣魚**]。

4. 在 [**反網路釣魚**或**ATP 反網路釣魚**] 頁面上, 執行下列其中一項操作:

    - 若要新增原則, 請選取 [ **+ 建立**]。
    - 若要編輯現有的原則, 請從 [**反網路釣魚**] 頁面上顯示的清單中選取原則名稱。 (或者, 您也可以在清單上方或選擇 [**預設原則**]。)在出現的頁面上, 選擇 [**編輯原則**]。  

5. 指定原則的名稱、描述及設定。 如需詳細資訊, 請參閱[瞭解 ATP 反網路釣魚原則選項](#learn-about-atp-anti-phishing-policy-options)。 

6. 檢查您的設定之後, 請選擇 [**建立這個原則**(或**儲存**)]。

## <a name="learn-about-atp-anti-phishing-policy-options"></a>瞭解 ATP 反網路釣魚原則選項

當您設定或編輯 ATP 反網路釣魚原則時, 您可以選擇數個選項, 以提供最複雜和全面的保護, 如下表所述:
  
|**此設定**|**執行動作**|**當您想要時, 請使用:**|
|:-----|:-----|:-----|
|**新增要保護的使用者** <br/> |定義將由原則保護的電子郵件地址。 您最多可以新增60個內部和外部地址, 以防止模擬模擬。  <br/> |當您想要確定來自您組織外部的郵件不是您正在保護之使用者清單上的其中一個使用者的模擬。 您可能想要保護的使用者範例包括高層級的主管、企業擁有者、外部董事會成員等等。  <br/> 此受保護的使用者清單與原則套用的人員清單不同, 而不是強制執行原則。 您可以在 [原則選項] 的 [套用**至**] 區段中, 定義 [套用至] 清單。  <br/> 例如, 如果您新增`Mary Smith <marys@contoso.com>`為要保護的使用者, 然後將原則套用至群組「所有使用者」。 這可確保已傳送至 "Mary" 群組中之使用者的郵件, 會由原則進行處理。  <br/> |
|**新增要保護的網域** <br/> |可讓您選擇要防止模擬的網域。 您可以指定原則包含所有自訂網域、逗號分隔的網域清單, 或兩者的組合。 如果您選擇 [**自動包含我擁有的網域**], 且後來將網域新增至您的 Office 365 組織, 則此反網路釣魚原則將會成為新網域的位置。  <br/> |當您想要確保來自您組織外部的郵件不會模擬您的已驗證網域清單中所定義的其中一個網域, 或是協力廠商網域。  <br/> |
|**選擇動作** <br/> |當 Office 365 偵測到您新增至原則的使用者和網域的模擬嘗試時, 請選擇要採取的動作。 您可以在相同的反網路釣魚原則中, 為使用者和網域選擇不同的動作。 這些動作適用于已由 Office 365 識別的任何內送電子郵件, 作為模擬此反網路釣魚原則保護的使用者帳戶或網域。  <br/> **隔離郵件**電子郵件會傳送至 Office 365 隔離區。 當您選擇此選項時, 不會將電子郵件傳送給原始收件者。  <br/> **將郵件重新導向至另一個電子郵件地址**電子郵件會傳送至您指定的電子郵件地址。 您可以指定多個電子郵件地址。 當您選擇此選項時, 不會將電子郵件傳送給原始收件者。  <br/> **將郵件移至收件者的 [垃圾郵件] 資料夾**電子郵件會傳送至收件者的 [垃圾郵件] 資料夾。 當您選擇此選項時, 電子郵件仍會傳送給原始收件者, 但不會放在收件者的收件匣中。  <br/> **傳遞郵件, 並將其他位址新增至 [密件副本] 行**電子郵件會傳送給原始收件者。 此外, 您所識別的使用者會在郵件的 [密件副本] 行傳遞之前加入。 當您選擇此選項時, 電子郵件仍會傳送給原始收件者的收件匣。  <br/> **不要套用任何動作**電子郵件會傳遞至原始收件者的收件匣。 不會對電子郵件訊息採取其他動作。  <br/> **開啟網路釣魚防護秘訣**在電子郵件中啟用反網路釣魚安全性提示。  <br/> |當您想要對 Office 365 所決定成為使用者或網域的模擬時, 在該原則中對郵件採取動作。  <br/> |
|**啟用信箱智慧** <br/> |啟用或停用此原則的信箱智慧。 您只能為雲端式帳戶啟用信箱智慧, 亦即, 信箱是完全裝載于 Office 365 的帳戶。  <br/> |當您想要根據每個使用者的個別寄件者地圖來增強使用者的模擬結果。 信箱智慧是圍繞您傳送和接收郵件的人員所建立。 這項智慧可讓 Office 365 自訂使用者層級的模擬原則, 以便更好地處理假肯定結果。  <br/> |
|**新增信任的寄件者和網域** <br/> |定義此原則不會視為 impersonations 的電子郵件地址和網域。 從寄件者電子郵件地址和您新增為信任的寄件者和網域的網域中的郵件, 都不會被分類為模擬型攻擊。 因此, 此原則中的動作和設定不會套用至來自這些寄件者和網域的郵件。  <br/> |當使用者與觸發模擬但被視為安全的網域或使用者互動時。 例如, 如果夥伴的顯示名稱或功能變數名稱與清單上定義的使用者相同。  <br/> |
|**套用至** <br/> |定義其傳入電子郵件會受限於原則規則的收件者。 您可以為與該原則相關聯的收件者建立條件和例外狀況。  <br/> 例如, 您可以將規則套用至網域中的所有收件者, 為您的組織建立全域原則。  <br/> 您也可以建立例外規則, 例如不會掃描特定收件者群組之電子郵件的規則。  <br/> |每個原則都必須與一組使用者相關聯, 例如特定群組或網域中的使用者。  <br/> |
|**高級網路釣魚閾值** <br/> |定義如何處理網路釣魚郵件的設定層級。  <br/> **Standard**置疑網路釣魚的電子郵件是以標準方式處理。  <br/> **主動**以高或非常高的置信度來處理電子郵件釣魚者的電子郵件, 會以相同的方式來處理。  <br/> **更積極**可疑的電子郵件是以中等、高或非常高的置信度, 以相同的方式處理。  <br/> **最主動**系統會以相同的方式處理具有低、中、高或非常高的網路釣魚詐騙的電子郵件。  <br/> |當您想要更積極地處理 Office 365 中可能的網路釣魚郵件。 例如, 發件者非常高的郵件可能會對其執行最嚴格的動作, 而不會對這些郵件採取較低的動作。 此設定也會影響篩選系統的其他部分, 以將信號組合在一起。 隨著設定層級的增加, 移動良好訊息的機率會增加。  <br/>|

|**此設定**|**執行動作**|**當您想要時, 請使用:**|
|:-----|:-----|:-----|

在您的組織設定反網路釣魚原則或 ATP 反網路釣魚原則之後, 您可以透過[查看高級威脅防護報告](view-reports-for-atp.md), 來瞭解服務的運作方式。
  
## <a name="example-anti-phishing-policy-to-protect-a-user-and-a-domain"></a>範例: 反網路釣魚原則來保護使用者和網域

此範例會設定名為「網域和 CEO」的原則, 此原則會提供使用者和網域的模擬防護, 然後將該原則套用至網域`contoso.com`中使用者所接收的所有電子郵件。 安全性管理員已決定原則必須符合這些商務需求:
  
- 原則需要為 CEO 的電子郵件帳戶和整個網域提供保護。

- 決定要對 CEO 使用者帳戶進行模擬嘗試的郵件, 必須重新導向至安全性系統管理員的電子郵件地址。

- 決定要對網域進行模擬嘗試的郵件會比較不緊急, 應該予以隔離, 供日後查看。

Contoso 的安全性系統管理員可能會使用類似下列的值, 來建立符合這些需求的反網路釣魚原則。
  
|||
|:-----|:-----|
|**設定或選項** <br/> |**範例** <br/> |
|名稱  <br/> |網域和 CEO  <br/> |
|描述  <br/> |確定 CEO 和我們的網域並未進行類比。  <br/> |
|新增要保護的使用者  <br/> |CEO 的電子郵件地址 (最少)。  <br/> |
|新增要保護的網域  <br/> |包含 CEO 之辦公室的組織網域。  <br/> |
|選擇動作  <br/> |如果由模擬的使用者傳送電子郵件: 選擇 [**將郵件重新導向至另一個電子郵件地址**], 然後輸入安全性系統管理員的電子`securityadmin@contoso.com`郵寄地址, 例如,。  <br/> 如果由模擬的網域傳送電子郵件: 選擇 [**隔離郵件**]。  <br/> |
|信箱智慧  <br/> |根據預設, 當您建立新的反網路釣魚原則時, 會選取信箱智慧。 將此設定保留為 [**開啟**] 以取得最佳結果。  <br/> |
|新增信任的寄件者和網域  <br/> |在這個範例中, 不要定義任何覆寫。  <br/> |
|套用至  <br/> |選取 **[收件**者] 網域。 在**其中任何一項**下, 選取 **[選擇**]。 選取 [ **+ 新增**]。 選取功能變數名稱旁的核取方塊, 例如`contoso.com`, 在清單中, 然後選取 [**新增**]。 選取 [**完成**]。  <br/> |

## <a name="delete-an-anti-phishing-or-atp-anti-phishing-policy"></a>刪除反網路釣魚或 ATP 反網路釣魚原則

您可以使用安全性&amp;與合規性中心來刪除您所建立的自訂原則。 您不能刪除組織的預設原則。 我們建議使用安全性&amp;與合規性中心來檢查或編輯任何 ATP 原則。
  
1. 移至[https://protection.office.com](https://protection.office.com) , 並使用您的公司或學校帳戶登入。

2. 在左側導覽中的 [**威脅管理**] 下, 選擇 [**原則**]。

3. 在 [**原則**] 頁面上, 選擇 [**反網路釣魚**] 或 [ **ATP 反網路釣魚**]。

4. 在 [**反網路釣魚**或**ATP 反網路釣魚**] 頁面上, 從清單中選取原則名稱。

5. 在出現的頁面上, 選擇 [**刪除原則**]。 允許最多30分鐘, 以將您的變更傳播至所有 Office 365 資料中心。
