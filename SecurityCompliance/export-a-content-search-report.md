# <a name="export-a-content-search-report"></a>匯出內容搜尋報告

而不是匯出搜尋結果一組完整的安全性 & 合規性中心中的內容搜尋從 （和內容搜尋與 eDiscovery 案例相關聯），您可以只匯出您匯出搜尋時，要產生相同報告結果。
  
當您匯出報告時，它會下載至具有相同名稱內容搜尋，但，後面會加 *_ReportsOnly*資料夾。 例如，如果內容搜尋名稱為*ContosoCase0815* ，報表會下載至名為*ContosoCase0815_ReportsOnly*的資料夾。 報告中包含的文件清單，請參閱[在報告中包含的內容](#whats-included-in-the-report)。

## <a name="before-you-begin"></a>開始之前

- 若要匯出內容搜尋報告，您必須獲指派 「 安全性 & 合規性中心中的符合性搜尋管理角色。 此角色指派至內建的 eDiscovery 管理員與組織管理角色群組。 它不是由預設指派給 「 組織管理 」 角色群組。 如需詳細資訊，請參閱[指派 eDiscovery 權限](assign-ediscovery-permissions.md)。
    
- 當您匯出報告時，資料會暫時儲存在 Microsoft cloud 中的唯一 Windows Azure 存放區域之前就會下載到本機電腦。 確定您的組織可以連線到 Azure，也就是在端點**\*。 blob.core.windows.net** （萬用字元代表您匯出的唯一識別碼）。 建立後的兩週時，會從 Azure 存放區刪除搜尋結果資料。 
    
- 您用來匯出搜尋結果的電腦必須符合下列系統需求：
    
  - 32 位元或 64 位元版本的 Windows 7 和更新版本
    
  - Microsoft.NET Framework 4.7
    
  - 支援的瀏覽器：
    
    - Microsoft Edge
    
      或
    
    - Microsoft Internet Explorer 10 或更新版本
    
    **附註：** Microsoft 不會製造廠商分機或 ClickOnce 應用程式的附加元件。 不支援匯出不受支援的瀏覽器使用協力廠商分機或附加元件的搜尋結果。 

- 如果內容搜尋所傳回的結果估計總大小超過 20 個&nbsp;TB，匯出報告將會失敗。 若要成功匯出報告，請嘗試以縮小範圍，然後重新執行搜尋，因此評估的結果的大小會小於 20&nbsp;TB。

- 匯出內容搜尋報告針對匯出執行在同一時間和匯出單一使用者可以執行的最大數目的最大數目的計數。 如需匯出限制的詳細資訊，請參閱[匯出內容搜尋結果](export-search-results.md#export-limits)。

## <a name="generate-and-download-a-content-search-report"></a>產生與下載內容搜尋報告

若要產生並下載內容搜尋報告的步驟是非常類似於實際匯出搜尋結果。
  
## <a name="step-1-generate-the-report-for-export"></a>步驟 1： 產生匯出的報告

第一個步驟是準備下載至您的電腦匯出報告。 當您報表，報表文件上傳至 Azure 存放裝置區域中的 microsoft 雲端。
  
1. 移至 [https://protection.office.com](https://protection.office.com)。
    
2. 使用公司或學校帳戶登入 Office 365。
    
3. 在 [安全性 & 合規性中心的左窗格中，按一下 [**搜尋** \> **內容搜尋**。
    
4. 在 [**內容搜尋**] 頁面上，選取搜尋。 
    
5. 在 [詳細資料] 窗格中，[**匯出至電腦的報告**，請按一下 [**產生報表**。
    
    > [!NOTE]
    > 如果搜尋結果超過 7 天，系統會提示您更新搜尋結果。 如果發生這種情況，取消匯出、 選取搜尋詳細資料窗格中按一下 [**更新搜尋結果**，並更新結果之後再開始執行報表匯出。 
  
6. 在**匯出報告**] 頁面的 [**包含從搜尋這些項目**，請選擇下列選項之一：
    
    - 匯出已編製索引的項目
    
    - 匯出已編製索引和未編製索引的項目
    
    - 匯出未編製索引的項目
    
    如需未編製索引的項目的詳細資訊，請參閱 <<c0>已局部編製索引內容搜尋中的項目。
    
7. 選擇要包含的所有版本的 SharePoint 文件的搜尋統計資料。 搜尋的內容來源包含 SharePoint 或 OneDrive for Business 網站時，才會出現這個選項。
    
8. 按一下 [**產生報告**。
    
    搜尋結果報告，已備妥下載，這表示報表文件將會上傳至 Microsoft 雲端中的 [Azure 儲存體] 區域。 準備好進行下載報告時，**下載報告**] 連結會在 [詳細資料] 窗格中顯示下**匯出報告到電腦**。 
    
> [!NOTE]
> 您也可以匯出報告與 eDiscovery 案例相關聯的內容搜尋。 若要這麼做，請前往 [ **eDiscovery** \> **eDiscovery**]，選取情況下，然後按一下 [**編輯**![編輯圖示](media/ebd260e4-3556-4fb0-b0bb-cc489773042c.gif)。 在 [**搜尋**] 頁面上，選取 [搜尋]，然後按一下 [**匯出**![匯出搜尋結果圖示](media/47205c65-babd-4b3a-bd7b-98dfd92883ba.png) \> **匯出報告**。 
  
## <a name="step-2-download-the-report"></a>步驟 2： 下載報告

下一步是到本機電腦的 Azure 儲存體區域下載報告。
  
1. 在您產生的報告，在 [**匯出至電腦的報告**] 下的搜尋詳細資料窗格中，按一下 [**下載報告**]。
    
    [**下載報告**] 頁面上會顯示，並包含下列資訊到該報表下載到您的電腦。 
    
    - 將下載的項目數。
    
    - 預估的大小總計將下載的項目。
    
    - 將匯出是否編製索引，或未建立索引。 未編製索引的項目是具有可辨識的格式、 加密，或因為其他原因而未建立索引的項目。
    
    - 是否會下載 SharePoint 文件的版本。
    
    - 報表匯出程序的狀態。 您可以開始下載報告，即使尚未完成要求報告的準備工作。
    
2. **匯出金鑰**，請按一下 [**複製到剪貼簿**。 若要下載報告，您將在步驟 5 中使用此機碼。
    
    > [!IMPORTANT]
    > 因為任何人都可以安裝和啟動 eDiscovery 匯出工具]，然後使用此機碼若要下載搜尋報告，請務必採取預防措施來保護此機碼，就像您會保護密碼或其他安全性相關的資訊。 
  
3. 按一下 [**下載報告**]。
    
4. 如果系統提示您安裝**MicrosoftOffice 365 eDiscovery 匯出工具**，請按一下 [**安裝**]。
    
5. 在**eDiscovery 匯出工具**中，貼上您在步驟 2 中適當的方塊中複製的匯出金鑰。
    
6. 按一下 [**瀏覽]** 以指定您要下載報告的位置。 
    
7. 按一下 [開始]**** 將搜尋結果下載至您的電腦。 
    
    **EDiscovery 匯出工具**會顯示在匯出程序，包括的估計項目數量 （及大小） 的其餘項目若要下載的狀態資訊。 匯出程序完成時，您可以存取已下載的所在位置的檔案。 
    
> [!NOTE]
> 您可以下載報告與 eDiscovery 案例相關聯的內容搜尋。 若要這麼做，請前往 [ **eDiscovery** \> **eDiscovery**]，選取情況下，然後按一下 [**編輯**![編輯圖示](media/ebd260e4-3556-4fb0-b0bb-cc489773042c.gif)。 在 [**匯出**] 頁面上，選取報表匯出，，然後按一下 [詳細資料窗格中的 [**下載報告**。 
  
## <a name="whats-included-in-the-report"></a>在報告中包含的內容

當產生並匯出內容搜尋結果的相關報告時，下載下列文件：
  
- **匯出摘要**-包含在匯出摘要的 Excel 文件。 這包括所搜尋的內容來源數目、 每個內容的位置、 估計的項目數目、 實際數目會匯出的項目從搜尋結果數目以及估計與實際的項目大小等資訊會匯出的。 
    
    > [!NOTE]
    > 匯出報告時，您就會包含未編製索引的項目，未編製索引的項目數目是否包含下載的搜尋結果 （如果您是要匯出搜尋結果） 中所列的總數和總數預估的搜尋結果中匯出摘要報告。 換句話說，也會下載的項目數總計等於總數預估結果及未編製索引的項目總數。 
  
- **資訊清單**的資訊清單中的檔案 （XML 格式），其中包含搜尋結果中包含每個項目的相關資訊。 
    
- **結果**-Excel 文件，其中包含具有會在搜尋結果匯出每個已編製索引項目的相關資訊的資料列。 電子郵件，結果記錄檔會包含每個郵件的資訊包括： 
    
  - 來源信箱中的郵件的位置 (包括郵件是否處於主要或封存信箱)。
    
  - 郵件已傳送或接收的日期。
    
  - 從郵件主旨行。
    
  - 寄件者和收件者的郵件。
    
    文件的 SharePoint 和 OneDrive for Business 網站，結果記錄檔包含每個文件的資訊包括：
    
  - 文件的 URL。
    
  - 文件所在之網站集合的 URL。
    
  - 上次修改文件的日期。
    
  - （這位於結果記錄檔中的 [主旨] 欄） 的文件的名稱。
    
    > [!NOTE]
    > **結果**報告中的資料列數目必須等於會列在**未編製索引的項目**報告中的項目總數減下載的搜尋結果的總數。 
  
- **未編製索引的項目**-Excel 文件，其中包含未編製索引的項目會包含在搜尋結果的相關資訊。 如果您未包含未編製索引的項目，在產生搜尋結果報告時，這份報告仍會被下載時，但會是空的。
