## <a name="just-in-time-access"></a><span data-ttu-id="997ad-132">即時存取</span><span class="sxs-lookup"><span data-stu-id="997ad-132">Just-in-Time Access</span></span>

<span data-ttu-id="997ad-133">Microsoft 會使用 Office 365 的即時 (JIT) 存取原則, 以緩解認證篡改風險和側向攻擊。</span><span class="sxs-lookup"><span data-stu-id="997ad-133">Microsoft uses the just-in-time (JIT) access principle for Office 365 to mitigate credential tampering risks and lateral attacks.</span></span> <span data-ttu-id="997ad-134">JIT 會移除服務的持續式系統管理存取, 並將權利取代為根據需求提升至這些角色的能力。</span><span class="sxs-lookup"><span data-stu-id="997ad-134">JIT removes persistent administrative access to services and replaces entitlements with the ability to elevate into those roles on demand.</span></span> <span data-ttu-id="997ad-135">從系統管理員移除持續性存取權限, 可確保認證只有在需要時才可供使用, 並降低認證盜竊風險。</span><span class="sxs-lookup"><span data-stu-id="997ad-135">Removing persistent access rights from administrators ensures credentials are available only when they are needed and reduces credential theft risks.</span></span>

<span data-ttu-id="997ad-136">JIT 存取模型要求工程師要求更高的許可權, 以執行系統管理責任。</span><span class="sxs-lookup"><span data-stu-id="997ad-136">The JIT access model requires engineers to request elevated privileges for a limited period to perform administrative duties.</span></span> <span data-ttu-id="997ad-137">此外, 工程師會使用以機器產生的複雜密碼建立的暫時帳戶, 並僅授與允許他們執行必要工作的角色。</span><span class="sxs-lookup"><span data-stu-id="997ad-137">In addition, engineers use temporary accounts created with machine-generated complex passwords and granted only those roles that allow them to perform necessary tasks.</span></span> <span data-ttu-id="997ad-138">例如, 由密碼箱所授與的系統管理存取是時間限制的, 而且所授與的存取時間取決於要求的角色。</span><span class="sxs-lookup"><span data-stu-id="997ad-138">For example, administrative access granted by Lockbox is time-bound, and the amount of time access granted depends on the requested role.</span></span> <span data-ttu-id="997ad-139">工程師指定要求存取密碼箱系統所需的時間。</span><span class="sxs-lookup"><span data-stu-id="997ad-139">An engineer specifies the duration of time access needed in the request to the Lockbox system.</span></span> <span data-ttu-id="997ad-140">當要求的時間超過提升的最大允許時間時, 加密箱系統會拒絕要求。</span><span class="sxs-lookup"><span data-stu-id="997ad-140">The Lockbox system rejects requests when time requested exceeds the maximum permitted time for the elevation.</span></span> <span data-ttu-id="997ad-141">到期之後, 系統會移除系統管理存取權, 而暫時帳戶就會過期。</span><span class="sxs-lookup"><span data-stu-id="997ad-141">After expiration, administrative access is removed and the temporary account expires.</span></span>

<span data-ttu-id="997ad-142">當授權和核准存取時, 工程師會收到由授權系統所產生的一次性管理密碼。</span><span class="sxs-lookup"><span data-stu-id="997ad-142">When authorized and approved for access, engineers receive a one-time use administrative password generated by the authorization system.</span></span> <span data-ttu-id="997ad-143">每次核准存取許可權要求時, 都會產生新密碼。</span><span class="sxs-lookup"><span data-stu-id="997ad-143">New passwords are generated each time a request for elevated access is approved.</span></span> <span data-ttu-id="997ad-144">密碼會被覆制到密碼安全, 並與工程師的 Microsoft 公司環境認證分開, 而且只適用于已核准的較高存取會話。</span><span class="sxs-lookup"><span data-stu-id="997ad-144">The password is copied into a password safe, is separate from the engineer's credentials for the Microsoft corporate environment, and is good only for the approved elevated access session.</span></span>

<span data-ttu-id="997ad-148">Office 365 伺服器會將同時會話限制為每個服務小組管理員、每個伺服器上的一個會話。</span><span class="sxs-lookup"><span data-stu-id="997ad-148">Office 365 servers restrict concurrent sessions to one session per-service team administrator, per-server.</span></span> <span data-ttu-id="997ad-149">TSGs 只允許使用者使用單一併行會話, 且不允許多個會話。</span><span class="sxs-lookup"><span data-stu-id="997ad-149">TSGs allow only a single concurrent session for users and do not allow multiple sessions.</span></span> <span data-ttu-id="997ad-150">TSGs 允許 Office 365 服務小組管理員同時連線至多部伺服器, 讓系統管理員可以有效地執行其職責。</span><span class="sxs-lookup"><span data-stu-id="997ad-150">Using a single session per server, TSGs allow Office 365 service team administrators to connect to multiple servers concurrently so administrators can effectively perform their duties.</span></span> <span data-ttu-id="997ad-151">服務小組管理員不具備 TSGs 本身的任何許可權。</span><span class="sxs-lookup"><span data-stu-id="997ad-151">Service team administrators do not have any permissions on the TSGs themselves.</span></span> <span data-ttu-id="997ad-152">TSG 僅用於強制執行多重要素驗證 (MFA) 和加密需求。</span><span class="sxs-lookup"><span data-stu-id="997ad-152">The TSG is used only to enforce multi-factor authentication (MFA) and encryption requirements.</span></span> <span data-ttu-id="997ad-153">一旦服務小組管理員透過 TSG 連線至特定伺服器, 特定伺服器就會強制執行每個系統管理員的會話限制。</span><span class="sxs-lookup"><span data-stu-id="997ad-153">Once the service team administrator connects to a specific server through a TSG, the specific server enforces a session limit of one per administrator.</span></span>
