# <a name="send-email-notifications-and-show-policy-tips-for-dlp-policies"></a>針對 DLP 原則傳送電子郵件通知並顯示原則提示

您可以使用資料遺失防護 (DLP) 原則來識別、監視和保護 Office 365 之間的機密資訊。 您想要讓組織中使用此敏感資訊的人員遵守您的 DLP 原則, 但您不想要封鎖它們, 就無法讓他們完成工作。 這是電子郵件通知和原則提示可以協助的地方。
  
![訊息列會顯示 Excel 2016 中的原則提示](media/7002ff54-1656-4a6c-993f-37427d6508c8.png)
  
原則提示是當有人使用與 DLP 原則衝突的內容時出現的通知或警告, 例如, 像是個人身分識別資訊 (PII) 的商務用 OneDrive 網站上的 Excel 活頁簿等內容, 也就是與外部使用者共用。
  
您可以使用電子郵件通知和原則提示來增加知名度, 並協助人們瞭解組織的原則。 您也可以讓人員選擇覆寫原則, 讓他們在有有效的業務需求或原則偵測到假肯定時, 不會封鎖。
  
在 Office 365 安全&amp;規範中心中, 當您建立 DLP 原則時, 您可以將使用者通知設定為:
  
- 傳送電子郵件通知給您選擇以描述問題的人員。
    
- 顯示與 DLP 原則衝突之內容的原則提示:
    
  - 針對 Outlook 網頁版和 Outlook 2013 及更新版本中的電子郵件, 在撰寫郵件時, 郵件的頂端會出現原則提示。
    
  - 對於商務用 OneDrive 帳戶或 SharePoint Online 網站中的檔, 原則提示會以出現在專案上的警告圖示來表示。 若要查看詳細資訊, 您可以選取專案, 然後在**** ![頁面的右上角](media/50b6d51b-92b4-4c5f-bb4b-4ca2d4aa3d04.png)選擇 [資訊資訊窗格] 圖示, 以開啟 [詳細資料] 窗格。 
    
  - 如果是儲存在 DLP 原則所包含的商務用 OneDrive 網站或 SharePoint Online 網站上的 Excel、PowerPoint 及 Word 檔, 則原則提示會出現在訊息列和 Backstage 檢視 ([檔案] 功能表**** \> **資訊**) 上。
    
## <a name="add-user-notifications-to-a-dlp-policy"></a>將使用者通知新增至 DLP 原則

當您建立 DLP 原則時, 電子郵件通知和原則提示都會是 [**使用者通知**] 一節中的一部分。 
  
1. 移至 [https://protection.office.com](https://protection.office.com)。
    
2. 使用公司或學校帳戶登入 Office 365。 您現在是在 Office 365 安全性&amp;與合規性中心。
    
3. 在&amp;安全性與合規性\>中心左側導覽\> **資料遺失防護** \> **原則** \> **+ 建立原則**。
    
    ![[建立原則] 按鈕](media/b1e48a08-92e2-47ca-abdc-4341694ddc7c.png)
  
4. 選擇保護您\> **接下來**所需之敏感資訊類型的 DLP 原則範本。
    
    若要以空白範本開始, 請選擇 [**自** \> **定義原則** \> **] [下一步]**。
    
5. 將原則\>命名為 **[下一步]**。
    
6. 若要選擇您想要讓 DLP 原則保護的位置, 請執行下列其中一項操作:
    
  - 選擇 [ \> **下一步]****的 Office 365 中的所有位置**。
    
  - 選擇 [**下一步]**。 **** \>
    
    若要包含或排除整個位置 (例如所有 Exchange 電子郵件或所有 OneDrive 帳戶), 請開啟或關閉該位置的**狀態**。 
    
    若要只包含特定的 SharePoint 網站或 OneDrive 帳戶, 請將**狀態**切換為 [開啟], 然後按一下 [**包含**] 下的連結, 選擇 [特定網站] 或 [帳戶]。 
    
7. 選擇 [**使用高級設定** \> **] [下一步]**。
    
8. 選擇 [ **+ 新增規則**]。
    
9. 在規則編輯器的 [**使用者通知**] 下, 切換 [開啟] 狀態。
    
    ![規則編輯器的使用者通知區段](media/47705927-c60b-4054-a072-ab914f33d15d.png)

> [!NOTE]
> DLP 原則適用于符合原則的所有檔, 不論這些檔是新的或現有的。 不過, 只有當新的內容符合現有的 DLP 原則時, 才會產生電子郵件通知。 現有的內容受到保護, 但不會透過電子郵件產生使用者通知。
  
## <a name="options-for-configuring-email-notifications"></a>設定電子郵件通知的選項

對於 DLP 原則中的每個規則, 您可以:
  
- 將通知傳送給您選擇的人員。 這些人員可以包含內容的擁有者、上次修改內容的人員、儲存內容的網站擁有者, 或特定使用者。
    
- 使用 HTML 或標記自訂通知中所包含的文字。 如需詳細資訊, 請參閱下文的章節。
    
> [!NOTE]
>  電子郵件通知只能傳送給個別收件者, 而不是群組或通訊群組清單。 只有新的內容會觸發電子郵件通知。 編輯現有的內容將會觸發原則提示, 但不會觸發電子郵件通知。 
  
![電子郵件通知選項](media/4e7b9500-2a78-44e6-9067-09f4bfd50301.png)
  
### <a name="default-email-notification"></a>預設電子郵件通知

通知具有以所採取動作為開頭的主旨行, 例如「通知」、「郵件被封鎖」, 或針對檔的「拒絕存取」。 如果通知是關於檔, 通知訊息內文會包含連結, 會帶您前往儲存檔的網站, 並開啟檔的原則提示, 您可以在其中解決任何問題 (請參閱下一節關於原則提示)。 如果通知是關於郵件, 通知會將符合 DLP 原則的郵件加入附件。
  
![通知訊息](media/35813d40-5fd8-425f-9624-55655e74fa6b.png)
  
根據預設, 通知會針對網站上的專案顯示類似下列的文字。 每個規則都會分別設定通知文字, 因此顯示的文字會因相符的規則而有所不同。

|**如果 DLP 原則規則執行此動作 .。。**|**如果是 SharePoint 或商務用 OneDrive 檔的預設通知, 就會指出 .。。**|**然後, Outlook 郵件的預設通知會指出 .。。**|
|:-----|:-----|:-----|
|傳送通知, 但不允許覆寫  <br/> |此專案與您組織中的原則相衝突。  <br/> |您的電子郵件與您組織中的原則衝突。  <br/> |
|封鎖存取、傳送通知, 並允許覆寫  <br/> |此專案與您組織中的原則相衝突。 如果您未解決此衝突, 則可能會封鎖對此檔案的存取。  <br/> |您的電子郵件與您組織中的原則衝突。 郵件不會傳遞給所有收件者。  <br/> |
|封鎖存取並傳送通知  <br/> |此專案與您組織中的原則相衝突。 除了其擁有者、最後一個修改者和主要網站集合管理員之外, 所有人都可以存取此專案。  <br/> |您的電子郵件與您組織中的原則衝突。 郵件不會傳遞給所有收件者。  <br/> |
   
### <a name="custom-email-notification"></a>自訂電子郵件通知

您可以建立自訂電子郵件通知, 而不是傳送預設的電子郵件通知給您的使用者或系統管理員。 自訂電子郵件通知支援 HTML, 且具有5000個字元的限制。 您可以使用 HTML 在通知中包含圖像、格式及其他品牌。
  
您也可以使用下列權杖來協助自訂電子郵件通知。 這些 token 是由所傳送通知中的特定資訊所取代的變數。

|**標記**|**描述**|
|:-----|:-----|
|%% AppliedActions%%  <br/> |套用至內容的動作。  <br/> |
|%% ContentURL%%  <br/> |SharePoint Online 網站或商務用 OneDrive 網站上檔的 URL。  <br/> |
|%%MatchedConditions%%  <br/> |內容所符合的條件。 使用此權杖通知人員可能發生的內容問題。  <br/> |

## <a name="options-for-configuring-policy-tips"></a>設定原則提示的選項

對於 DLP 原則中的每個規則, 您可以設定原則提示:
  
- 只要通知人員內容與 DLP 原則衝突, 就能採取動作來解決衝突。 您可以使用預設文字 (請參閱下表) 或輸入您組織的特定原則的自訂文字。
    
- 允許使用者覆寫 DLP 原則。 您可以選擇:
    
  - 需要人員輸入業務理由以覆寫原則。 將會記錄此資訊, 您可以在安全性&amp;與合規性中心的 [**報告**] 區段中, 于 [DLP 報告] 中加以查看。 
    
  - 允許人員報告 false 正值, 並覆寫 DLP 原則。 此資訊也會記錄報告, 因此您可以使用假正值來微調您的規則。
    
![原則提示選項](media/0d2f2c68-028a-4900-afe6-1d9fce5303ef.png)
  
例如, 您可能會將 DLP 原則套用至商務用 OneDrive 網站, 以偵測個人身分識別資訊 (PII), 且此原則具有三個規則:
  
1. 第一個規則: 如果在檔中偵測到的這個敏感資訊的實例少於五個, 且該檔與組織內的人員共用, 則 [**傳送通知**] 動作會顯示原則提示。 針對原則提示, 不需要覆寫選項, 因為此規則只是通知人員, 而不會封鎖存取。 
    
2. 第二個規則: 如果在檔中偵測到此機密資訊有五個以上的實例, 且該檔與組織內的人員共用, 則 [**封鎖存取內容**] 動作會限制檔案的許可權, 而 **[傳送通知**] 動作可讓人員提供業務理由, 以覆寫此規則中的動作。 您組織的業務有時會要求內部人員共用 PII 資料, 而您不想讓 DLP 原則封鎖此工作。 
    
3. 第三個規則: 如果在檔中偵測到此敏感資訊有五個以上的實例, 而且該檔與組織外部的人員共用, 則 [**封鎖存取內容**] 動作會限制檔案的許可權, 而 **[傳送通知**] 動作不允許使用者覆寫此規則中的動作, 因為資訊是從外部共用。 在任何情況下, 您的組織中的人員都不應該允許共用組織外部的 PII 資料。 
    
以下是瞭解如何使用原則提示來覆寫規則的一些要點:
  
- 要覆寫的選項是每個規則, 而且會覆寫規則中的所有動作 (傳送無法覆寫的通知除外)。
    
- 內容可以比對 DLP 原則中的數個規則, 但只會顯示最嚴格的原則提示和最高優先順序的規則。 例如, 從規則來封鎖內容存取的原則提示, 會從只傳送通知的規則的原則提示中顯示。 這可防止人們看到層疊的原則提示。
    
- 如果最嚴格規則中的原則提示允許使用者覆寫規則, 則覆寫此規則也會覆寫內容符合的任何其他規則。
    
## <a name="policy-tips-on-onedrive-for-business-sites-and-sharepoint-online-sites"></a>商務用 OneDrive 網站和 SharePoint Online 網站上的原則提示

當商務用 OneDrive 網站或 SharePoint Online 網站上的檔符合 DLP 原則中的規則, 且該規則使用原則提示時, 原則提示會在檔上顯示特殊的圖示:
  
1. 如果規則傳送有關檔案的通知, 就會出現警告圖示。
    
2. 如果規則會封鎖對檔的存取, 會顯示封鎖的圖示。
    
![OneDrive 帳戶中檔的原則提示圖示](media/d3e9f772-03f9-4d28-82f8-3064784332a2.png)
  
若要在檔上採取動作, 您可以選取專案\>的右上角的 [](media/50b6d51b-92b4-4c5f-bb4b-4ca2d4aa3d04.png) **資訊** ![資訊窗格] 圖示, 以開啟 [詳細資料] 窗格的\> [**原則提示**]。
  
原則提示會列出內容的問題, 如果使用這些選項設定原則提示, 您可以選擇 [**解決**], 然後覆**寫**原則提示或**報告**false 正值。 
  
![顯示原則提示的資訊窗格](media/0a191e70-80f0-4702-90f4-7a5b7aabcaab.png)
  
![具有覆寫選項的原則提示](media/e250bff9-41d5-4ce4-82ea-1dc2d043fab1.png)
  
DLP 原則會同步處理至網站, 而且 contented 會定期或以非同步方式評估, 因此您在建立 DLP 原則的時間與開始查看原則提示的時間之間可能會有短暫的延遲。 當您解決或覆寫網站上檔上的圖示時, 可能會發生類似的延遲。
  
### <a name="default-text-for-policy-tips-on-sites"></a>網站上的原則提示的預設文字

根據預設, 原則提示會針對網站上的專案顯示類似下列的文字。 每個規則都會分別設定通知文字, 因此顯示的文字會因相符的規則而有所不同。

|**如果 DLP 原則規則執行此動作 .。。**|**預設原則提示會指出 .。。**|
|:-----|:-----|
|傳送通知, 但不允許覆寫  <br/> |此專案與您組織中的原則相衝突。  <br/> |
|封鎖存取、傳送通知, 並允許覆寫  <br/> |此專案與您組織中的原則相衝突。 如果您未解決此衝突, 則可能會封鎖對此檔案的存取。  <br/> |
|封鎖存取並傳送通知  <br/> |此專案與您組織中的原則相衝突。 除了其擁有者、最後一個修改者和主要網站集合管理員之外, 所有人都可以存取此專案。  <br/> |
   
### <a name="custom-text-for-policy-tips-on-sites"></a>網站上的原則提示的自訂文字

您可以從電子郵件通知自訂原則提示的文字。 不同于電子郵件通知的自訂文字 (請參閱上述章節), 原則提示的自訂文字不接受 HTML 或標記。 相反地, 原則提示的自訂文字是純文字, 僅含256字元的限制。
  
## <a name="policy-tips-in-outlook-on-the-web-and-outlook-2013-and-later"></a>Outlook 網頁版和 Outlook 2013 及更新版本中的原則提示

當您在網頁版 Outlook 和 Outlook 2013 和更新版本中撰寫新的電子郵件時, 如果您新增的內容符合 DLP 原則中的規則, 而該規則使用原則提示, 就會看到原則提示。 原則提示會出現在郵件上方的收件者上方, 同時撰寫郵件時。
  
![撰寫郵件頂端的原則提示](media/9b3b6b74-17c5-4562-82d5-d17ecaaa8d95.png)
  
原則提示可供敏感資訊顯示在郵件內文、主旨行, 甚至是郵件附件, 如下所示。
  
![顯示附件與 DLP 原則衝突的原則提示](media/59ae6655-215f-47d9-ad1d-39c0d1e61740.png)
  
如果原則提示設定為允許覆寫, 您可以選擇 [**顯示詳細資料** \> **** \> \> ** **]。
  
![郵件展開為顯示覆寫選項的原則提示](media/28bfb997-48a6-41f0-8682-d5e62488458a.png)
  
![[原則提示] 對話方塊, 您可以在其中覆寫原則提示](media/f97e836c-04bd-44b4-aec6-ed9526ea31f8.png)
  
請注意, 當您將機密資訊新增至電子郵件時, 在新增敏感資訊和顯示原則提示時, 可能會有延遲。

### <a name="outlook-2013-and-later-supports-showing-policy-tips-for-only-some-conditions"></a>Outlook 2013 和更新版本支援顯示僅限某些條件的原則提示

目前, Outlook 2013 和更新版本支援只顯示這些條件的原則提示:

- 內容包含
- 共用內容

我們目前正在處理支援, 以顯示其他條件的原則提示。 這些包括：

- 無法掃描任何電子郵件附件的內容
- 任何電子郵件附件的內容未完成掃描
- 附件副檔名為
- 附件受密碼保護
- Document 屬性為
- 收件者網域是
- 寄件者 IP 位址為

請注意, 所有這些條件都能在 Outlook 中運作, 在其中會比對內容並對內容強制執行保護動作。 但尚未支援使用者顯示原則提示。
  
### <a name="policy-tips-in-the-exchange-admin-center-vs-the-office-365-security-amp-compliance-center"></a>Exchange 系統管理中心與 Office 365 安全性&amp;與合規性中心中的原則提示

原則提示可以使用在 Exchange 系統管理中心建立的 DLP 原則和郵件流程規則, 或使用 Office 365 安全性&amp;與規範中心建立的 dlp 原則, 但不能同時使用這兩者。 這是因為這些原則會儲存在不同的位置, 但原則提示只能從單一位置進行繪製。
  
如果您已在 Exchange 系統管理中心中設定原則提示, 您在 Office 365 安全性&amp;與合規性中心中設定的任何原則提示, 將不會對 outlook 網頁版或 outlook 2013 和更新版本中的使用者顯示, 直到您關閉 Exchange 中的秘訣。系統管理中心。 這可確保您目前的 Exchange 郵件流程規則 (也稱為傳輸規則) 可繼續運作, 直到您選擇切換到 Office 365 安全性&amp;與合規性中心。
  
請注意, 雖然原則提示只能從單一位置進行繪製, 但即使您在 Office 365 安全性&amp;合規性中心和 Exchange 系統管理中心使用 DLP 原則, 仍會傳送電子郵件通知。
  
### <a name="default-text-for-policy-tips-in-email"></a>電子郵件中原則提示的預設文字

原則提示預設會顯示類似下列電子郵件的文字。

|**如果 DLP 原則規則執行此動作 .。。**|**預設原則提示會指出 .。。**|
|:-----|:-----|
|傳送通知, 但不允許覆寫  <br/> |您的電子郵件與您組織中的原則相衝突。  <br/> |
|封鎖存取、傳送通知, 並允許覆寫  <br/> |您的電子郵件與您組織中的原則相衝突。  <br/> |
|封鎖存取並傳送通知  <br/> |您的電子郵件與您組織中的原則相衝突。  <br/> |
   
## <a name="policy-tips-in-excel-powerpoint-and-word"></a>Excel、PowerPoint 和 Word 中的原則提示

當人員在 Excel、PowerPoint 及 Word 的桌上出版本中使用機密內容時, 原則提示可以即時通知內容與 DLP 原則相衝突。 這需要:
  
- Office 檔儲存在商務用 OneDrive 網站或 SharePoint Online 網站上。
    
- 此網站包含在設定為使用原則提示的 DLP 原則中。
    
Office 桌面程式會自動從 Office 365 自動同步處理 DLP 原則, 然後掃描您的檔, 以確保它們不會與您的 DLP 原則衝突, 並即時顯示原則提示。
  
根據您在 DLP 原則中設定原則提示的方式, 使用者可以選擇直接忽略原則提示、使用或不使用業務理由覆寫原則, 或報告為 false 正值。
  
原則提示會出現在訊息列上。
  
![訊息列會顯示 Excel 2016 中的原則提示](media/7002ff54-1656-4a6c-993f-37427d6508c8.png)
  
原則提示也會出現在 Backstage 檢視中 (在 [ **** 檔案] 索引標籤上)。 
  
![Backstage 會顯示 Excel 2016 中的原則提示](media/44c561f6-8f3f-4878-b1b0-b7543f8a4120.png)
  
如果已使用這些選項設定 DLP 原則中的原則提示, 您可以選擇 [**解決**], 以覆**寫**原則提示或**報告**false 正值。 
  
![Excel 2016 的 Backstage 中的原則提示選項](media/5b3857ba-907e-456e-ae43-888b594c049c.png)
  
在每個 Office 桌面程式中, 使用者可以選擇關閉原則提示。 如果關閉, 則簡單通知的原則提示將不會出現在訊息列或 Backstage 檢視上 (在 [ **** 檔案] 索引標籤上)。 不過, 有關封鎖和覆寫的原則提示仍會出現, 而且他們仍然會收到電子郵件通知。 此外, 關閉原則提示不會將檔從已套用的任何 DLP 原則中排除。 
  
### <a name="default-text-for-policy-tips-in-excel-2016-powerpoint-2016-and-word-2016"></a>Excel 2016、PowerPoint 2016 和 Word 2016 中的原則提示的預設文字

根據預設, 原則提示會在已開啟檔的訊息列和 Backstage 檢視上顯示類似下列的文字。 每個規則都會分別設定通知文字, 因此顯示的文字會因相符的規則而有所不同。

|**如果 DLP 原則規則執行此動作 .。。**|**預設原則提示會指出 .。。**|
|:-----|:-----|
|傳送通知, 但不允許覆寫  <br/> |此檔案與您組織中的原則相衝突。 如需詳細**** 資訊, 請移至 [檔案] 功能表。  <br/> |
|封鎖存取、傳送通知, 並允許覆寫  <br/> |此檔案與您組織中的原則相衝突。 如果您未解決此衝突, 則可能會封鎖對此檔案的存取。 如需詳細**** 資訊, 請移至 [檔案] 功能表。  <br/> |
|封鎖存取並傳送通知  <br/> |此檔案與您組織中的原則相衝突。 如果您未解決此衝突, 則可能會封鎖對此檔案的存取。 如需詳細**** 資訊, 請移至 [檔案] 功能表。  <br/> |
   
### <a name="custom-text-for-policy-tips-in-excel-powerpoint-and-word"></a>Excel、PowerPoint 和 Word 中的原則提示的自訂文字

您可以從電子郵件通知自訂原則提示的文字。 不同于電子郵件通知的自訂文字 (請參閱上述章節), 原則提示的自訂文字不接受 HTML 或標記。 相反地, 原則提示的自訂文字是純文字, 僅含256字元的限制。
