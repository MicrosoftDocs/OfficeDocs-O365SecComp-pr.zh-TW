# <a name="apply-information-rights-management-irm-to-a-list-or-library"></a>套用至清單或文件庫的資訊版權管理 (IRM)

您可以使用資訊版權管理 (IRM) 協助控制及保護從清單或文件庫下載的檔案。
  
## <a name="before-you-begin"></a>開始之前

- Azure 版權管理服務 (Azure RMS) 從 Azure 資訊保護及內部同等 Active Directory Rights Management Services (AD RMS) 支援資訊版權管理的網站。沒有不同或其他安裝所需。
    
- 將 IRM 套用至清單或文件庫之前它必須先啟用您的網站管理員。
    
- 若要套用至清單或文件庫的 IRM，您必須具備清單或文件庫的管理員權的限。
    
- 如果您使用 SharePoint Online，讓使用者下載較大的受 IRM 保護之檔案時可能會發生逾時。如果發生這種情況，然後使用您的 Office 程式套用 IRM 保護及較大的檔案儲存在不使用 IRM 的 SharePoint 文件庫。
    
> [!NOTE]
> 如果您使用 SharePoint Server 2013，伺服器管理員必須在所有前端網頁伺服器的每一個您組織中的人員想要使用 IRM 保護的檔案類型安裝保護裝置。

## <a name="what-is-information-rights-management"></a>資訊版權管理新功能
<a name="__toc256598175"> </a>

資訊版權管理 (IRM) 可讓您限制使用者可以對已從清單或文件庫下載的檔案採取的動作。IRM 加密下載的檔案並限制的使用者和允許解密這些檔案的程式]。IRM 也可以限制的權限的使用者可以讀取檔案，使其無法採取動作，例如列印份數的檔案或從這些複製文字。
  
您可以使用清單或文件庫上的 IRM 限制機密內容的散佈。例如，如果您建立文件庫與所選的行銷代表共用即將來臨的產品的相關資訊，您可以使用 IRM 防止這些個人共用此內容與其他公司內的員工。
  
在網站上，您會套用 IRM 整個清單或文件庫，而不是個別的檔案。這會使確保一致的保護層級的整個集的文件或檔案更容易。IRM 如此可協助您的組織來強制執行管理及散佈的機密或專屬資訊的公司原則。
  
> [!NOTE]
> 本頁內容資訊版權管理的相關資訊的任何 Microsoft SharePoint Server 2013 與 SharePoint Server 2016 授權字詞協議替代任何參照 「 資訊版權管理 」 的字詞。 
  
### <a name="how-irm-can-help-protect-content"></a>IRM 如何協助保護內容
<a name="__toc256598176"> </a>

IRM 可協助保護受限制的內容如下：
  
- 有助於防止從複製、 修改、 列印、 傳真、 或複製並貼上未經授權的使用的內容授權檢視器
    
- 若要防止使用 Print Screen 功能在 Microsoft Windows 中複製的內容已授權的檢視器可協助
    
- 若要防止未經授權的檢視器如果它從伺服器下載之後傳送電子郵件中檢視的內容可協助
    
- 在指定期間內的時間之後，使用者必須確認其認證並再次下載內容的內容會限制存取
    
- 協助強制執行公司原則的管理及散佈的組織內的內容
    
### <a name="how-irm-cannot-help-protect-content"></a>IRM 不能如何協助保護內容
<a name="__toc256598177"> </a>

IRM 無法防止受限制的內容下列：
  
- 清除、 竊取、 擷取或由如特洛伊木馬、 按鍵記錄器及間諜軟體特定類型的惡意程式傳輸
    
- 遺失或損毀因為電腦病毒的動作
    
- 手動複製或重新輸入的內容從螢幕顯示
    
- 數位或網片攝影的螢幕顯示的內容
    
- 透過協力廠商螢幕擷取程式複製
    
- 複製的內容中繼資料 （資料行值） 透過協力廠商螢幕擷取程式或複製並貼上巨集指令
    
[將資訊版權管理套用至清單或文件庫](https://support.office.com/article/6714cfe3-ef39-43b0-bb65-a887726bb63c)
  
## <a name="how-irm-works-for-lists-and-libraries"></a>IRM 清單和文件庫的運作方式
<a name="__toc256598178"> </a>

IRM 保護套用至清單或文件庫層級的檔案。當文件庫啟用 IRM 時，版權管理會套用於所有的文件庫中的檔案。當清單啟用 IRM 時，版權管理只適用於清單項目不實際的清單項目附加的檔案。
  
當人員下載中啟用 IRM 的清單或文件庫的檔案時，以便只授權的人員可檢視它們加密檔案。每個受版權管理檔案也包含會限制在檢視之檔案的人員的發行授權。一般限制包括唯讀屬性，停用的文字，防止人員從儲存的本機複本，並防止人員列印檔案複製進行檔案。可讀取的 IRM 支援的檔案類型的用戶端程式受版權管理檔案內使用的發行授權強制執行這些限制。這是如何受版權管理檔案會保留在其保護即使它從伺服器下載。
  
從清單或文件庫下載時套用至檔案的限制類型根據包含該檔案的網站上的個別使用者的權限。下表說明如何在網站上的權限對應至 IRM 權限。
  
|**權限**|**IRM 權限**|
|:-----|:-----|
|管理權限、 管理網站  <br/> |**完全控制**（如用戶端程式所定義）： 這個權限通常可讓使用者讀取、 編輯、 複製、 儲存及修改受版權管理內容的權限。  <br/> |
|編輯項目、 管理清單、 新增並自訂頁面  <br/> |**編輯**、**複製**及**儲存**： 使用者可以列印檔案只有當在清單或文件庫的資訊版權管理設定] 頁面上選取 [**允許使用者列印文件**] 核取方塊。  <br/> |
|檢視項目  <br/> |**讀取**： 使用者可以讀取文件，但無法複製或修改其內容。使用者可以列印只有當在清單或文件庫的資訊版權管理設定] 頁面上選取 [**允許使用者列印文件**] 核取方塊。<br/> |
|其他  <br/> |沒有其他權限會直接對應到 IRM 權限。  <br/> |
   
當您針對清單或文件庫在 SharePoint Server 2013 中的啟用 IRM 時，您可以僅保護清單或文件庫的所有前端網頁伺服器安裝保護裝置中的檔案類型。保護裝置是控制加密與解密的受版權管理的特定檔案格式的檔案計畫。SharePoint 包含下列檔案類型的保護裝置：
  
- Microsoft Office InfoPath 表單
    
- 下列的 Microsoft Office 程式 97-2003年檔案格式： Word、 Excel 及 PowerPoint
    
- Office Open XML 格式的下列的 Microsoft Office 程式： Word、 Excel 及 PowerPoint
    
- XML Paper Specification (XPS) 格式
    
如果您的組織計劃使用 IRM 保護除了上列以外的其他檔案類型]，伺服器管理員必須安裝保護裝置這些其他檔案格式。
