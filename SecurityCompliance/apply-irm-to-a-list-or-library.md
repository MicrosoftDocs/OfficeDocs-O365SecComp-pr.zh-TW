# <a name="apply-information-rights-management-irm-to-a-list-or-library"></a>將資訊版權管理 (IRM) 套用至清單或文件庫

您可以使用資訊版權管理 (IRM) 協助控制及保護下載從清單或文件庫的檔案。
  
## <a name="before-you-begin"></a>開始之前

- Azure 資訊保護，並在內部同等 Active Directory Rights Management Services (AD RMS)，從 Azure 版權管理服務 (Azure RMS) 支援資訊版權管理網站。 沒有不同或其他安裝所需。
    
- 您將 IRM 套用至清單或文件庫之前必須先啟用由系統管理員為您的網站。
    
- 若要將 IRM 套用至清單或文件庫，您必須使用該清單或文件庫的系統管理員權限。
    
- 如果您使用 SharePoint Online，您的使用者可能會下載較大受 IRM 保護的檔案時遇到逾時。 如果發生這種情況，然後使用您的 Office 程式，來套用 IRM 保護，並將較大的檔案儲存在不使用 IRM 的 SharePoint 文件庫。
    
> [!NOTE]
> 如果您使用 SharePoint Server 2013，伺服器管理員必須針對每個組織中的人員想要使用 IRM 保護的檔案類型的所有前端網頁伺服器上安裝保護裝置。

2. 功能區上，按一下 [**文件庫**] 索引標籤，然後按一下**文件庫設定**。 （如果您正在工作清單中，按一下 [**清單**] 索引標籤，然後按一下 [**清單設定**。）

## <a name="what-is-information-rights-management"></a>什麼是資訊版權管理？
<a name="__toc256598175"> </a>

資訊版權管理 (IRM) 可讓您以限制使用者可以在已從清單或文件庫下載的檔案採取的動作。 IRM 會加密下載的檔案，並限制能夠解密這些檔案的使用者及程式。 IRM 也會限制能夠讀取檔案的使用者權限，如此一來，使用者就無法進行像是列印檔案複本或從檔案複製文字之類的動作。
  
您可以在清單或文件庫中使用 IRM，來限制的敏感內容散佈。 例如，如果您要建立要與選取行銷人員共用資訊即將來臨的產品文件庫，您可以使用 IRM，防止這些人與其他員工在公司中共用此內容。
  
在網站上，您將 IRM 套用至整個清單或文件庫，而不是個別的檔案。 這有助於確保一致的整個集的文件或檔案的保護層級。 IRM 因此可以協助您的組織來強制執行公司控管的使用和散佈的專利或機密資訊的原則。
  
> [!NOTE]
> 資訊版權管理有關此頁面上的資訊會參照 '資訊版權管理' 任何字詞取代任何 Microsoft SharePoint Server 2013 和 SharePoint Server 2016 授權字詞協議中。 
  
### <a name="how-irm-can-help-protect-content"></a>IRM 可以如何協助保護內容
<a name="__toc256598176"> </a>

IRM 可協助保護受限制的內容如下：
  
- 這有助於避免複製、 修改、 列印、 傳真，或複製並貼上未經授權的使用的內容從授權檢視器
    
- 這有助於避免複製內容藉由使用 Microsoft Windows 中的 [列印螢幕] 功能已授權的檢視器
    
- 有助於防止未經授權的檢視器檢視的內容，如果它從伺服器下載之後，在電子郵件送出
    
- 限制存取權以一段時間的時間之後，使用者必須確認他們的認證，並再次下載內容的內容
    
- 協助強制執行公司使用與組織內的內容的散佈，進而管理的原則
    
### <a name="how-irm-cannot-help-protect-content"></a>IRM 無法如何協助保護內容
<a name="__toc256598177"> </a>

IRM 無法防止受限制的內容下列：
  
- 清除、 竊取、 擷取或由特洛伊木馬、 按鍵輸入記錄器等某些類型的間諜軟體的惡意程式的傳輸
    
- 遺失或損毀，因為電腦有病毒的動作
    
- 手動複製或重新輸入的內容從螢幕上顯示
    
- 數位或網片攝影的螢幕顯示的內容
    
- 透過協力廠商螢幕擷取程式複製
    
- 複製內容的中繼資料 （資料行的值） 透過協力廠商螢幕擷取程式或複製並貼上巨集指令

## <a name="how-irm-works-for-lists-and-libraries"></a>IRM 的清單和文件庫的運作方式
<a name="__toc256598178"> </a>

IRM 保護套用至清單或文件庫層級的檔案。 文件庫啟用 IRM 時，版權管理套用至所有該文件庫中的檔案。 如需清單啟用 IRM 時，版權管理僅適用於清單項目，不實際清單項目所附加的檔案。
  
當人員下載中啟用 IRM 的清單或文件庫的檔案時，檔案加密，因此只有授權的使用者可以檢視它們。 每個受版權管理檔案也包含會限制檢視檔案的人員的發行授權。 一般限制包括唯讀屬性，停用複製文字，防止人員從儲存本機複本，並列印檔案時，防止人員進行檔案。 可以讀取 IRM 支援的檔案類型的用戶端程式會使用受版權管理檔案內的發行授權，以強制執行這些限制。 這是如何受版權管理檔案會保留其保護即使它從伺服器下載。
  
它會從清單或文件庫下載時，會套用至檔案的限制的類型根據包含檔案的網站上的個別使用者的權限。 下表說明如何在網站上的權限對應至 IRM 權限。
  
|**權限**|**IRM 權限**|
|:-----|:-----|
|管理權限、 管理網站  <br/> |**完全控制**（如用戶端程式所定義）： 此權限通常可讓使用者讀取、 編輯、 複製、 儲存及修改受版權管理內容的權限。  <br/> |
|編輯項目、 管理清單、 新增並自訂頁面  <br/> |**編輯**、**複製**及**儲存**： 使用者可以列印的檔案，才**允許使用者列印文件**] 核取方塊已選取清單或文件庫的資訊版權管理設定] 頁面上。  <br/> |
|檢視項目  <br/> |**讀取**： 使用者可以讀取文件，但不能複製或修改其內容。 只有在清單或文件庫的資訊版權管理設定] 頁面上已選取 [**允許使用者列印文件**核取方塊，可以列印使用者。  <br/> |
|其他  <br/> |沒有其他權限會直接對應到 IRM 權限。  <br/> |
   
當您啟用 IRM 的清單或文件庫在 SharePoint Server 2013 中的時，您可以只會保護該清單或文件庫的所有前端網頁伺服器安裝保護裝置中的檔案類型。 保護裝置是控制項的加密和解密受版權管理特定的檔案格式的檔案的程式。 SharePoint 還包含下列檔案類型的保護裝置：
  
- Microsoft Office InfoPath 表單
    
- 下列的 Microsoft Office 程式的 97-2003年檔案格式： Word、 Excel 及 PowerPoint
    
- Office Open XML 格式的下列的 Microsoft Office 程式： Word、 Excel 及 PowerPoint
    
- XML Paper Specification (XPS) 格式
    
如果您的組織計劃要用於 IRM 保護除了上面所列的任何其他檔案類型，您的伺服器管理員必須安裝下列其他檔案格式的保護裝置。
