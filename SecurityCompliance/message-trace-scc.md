# <a name="message-trace-in-the-security--compliance-center"></a>安全性 & 規範中心中的郵件追蹤

安全性 & 規範中心中的郵件追蹤遵循出差透過 Exchange Online 組織的電子郵件。您可以決定如果已收到、 拒絕、 延遲、 或郵件傳遞服務。它也會顯示動作所採取的郵件上之前它達到其最終狀態。

在 Exchange 系統管理中心 (EAC) 中所提供的郵件追蹤時改善安全性 & 規範中心中的郵件追蹤。您可以使用郵件追蹤資訊有效率地回答關於何其訊息的使用者問題、 進行疑難排解郵件流程，及驗證原則的變更。

## <a name="open-message-trace"></a>開啟郵件追蹤

1. 使用您的 公司或學校帳戶[登入 Office 365](https://support.office.com/article/e9eb7d51-5430-4929-91ab-6157c5a050b4)。

2. ，然後選擇 [系統管理]。

3. 在左下導覽列中，依序展開 [**系統管理置中**並選取**安全性 & 規範**。

4. 在開啟的**安全性 & 規範**頁面上，依序展開 [**郵件流程**，並選取**郵件追蹤**。

### <a name="more-search-options"></a>更多搜尋選項

#### <a name="delivery-status"></a>傳遞狀態

您可以將保留**所有**選取的預設值或您可以選取其中一個來篩選結果的下列值：

- **已傳遞**： 郵件已成功傳遞至預定目的地。

- **擱置**： 正在郵件的傳送嘗試或重新嘗試。

- **Expanded**： 個別群組的成員在傳遞之前已展開的通訊群組收件者。

- **失敗**： 無法傳遞郵件。

- **隔離**： 郵件遭隔離時 （做為垃圾郵件、 大量郵件或網路釣魚）。如需詳細資訊，請參閱[Office 365 中的隔離電子郵件訊息](https://support.office.com/article/4c234874-015e-4768-8495-98fcccfc639b.aspx)。

- **為垃圾郵件篩選**： 郵件已識別垃圾郵件和已拒絕或封鎖的 （未隔離）。

- **取得狀態：** Office 365 最近接收到郵件但尚未可以使用任何其他狀態資料。存回幾分鐘的時間。

**請注意**：**擱置、** **隔離**，並**為垃圾郵件篩選**的值只適用於搜尋少於 10 天。此外，可能會有 5 到 10 分鐘的延遲之間的實際和報告的傳遞狀態。

#### <a name="message-id"></a>郵件識別碼

這是中找到的網際網路郵件識別碼 （也稱為用戶端識別碼）**訊息識別碼：** 郵件標頭中的標頭] 欄位。使用者可以提供此值設為調查特定的郵件。

這個值是郵件的存留期的常數。建立 Office 365 或 Exchange 中的郵件，值為格式`<GUID@ServerFQDN>`，包括角括弧 (\< \>)。例如， `<d9683b4c-127b-413a-ae2e-fa7dfb32c69d@DM3NAM06BG401.Eop-nam06.prod.protection.outlook.com>`。其他郵件系統可能會使用不同的語法或值。此值應該是唯一的但不是所有的電子郵件系統嚴格遵守下列需求。如果**訊息識別碼：** 、 標頭欄位不存在或是空白的內送郵件從外部來源的指派為任意值。

當您使用**訊息識別碼**來篩選結果時，請務必包含完整的字串，包括任何角括弧。

#### <a name="direction"></a>方向

### <a name="enhanced-summary-reports"></a>增強的摘要報告

- **sender_address 採用**： 寄件者的電子郵件地址 (*別名*@*網域*)。

- **Recipient_status**： 郵件的收件者傳送的狀態。如果郵件傳送給多個收件者，它會顯示所有收件者和對應的狀態格式的每個： \<*電子郵件地址*\>##\<*狀態*\>。例如：

- **message_subject**： 郵件的**主旨**] 欄位的前 256 個字元。

- **total_bytes**： 以位元組為單位，包括附件的郵件大小。

- **connector_id**： 來源或目的地連接器名稱。如需 Exchange Online 中的連接器的詳細資訊，請參閱[使用 Office 365 中的連接器設定郵件流程](https://docs.microsoft.com/Exchange/mail-flow-best-practices/use-connectors-to-configure-mail-flow/use-connectors-to-configure-mail-flow)。

- **delivery_priority**<sup>*</sup>： 是否將郵件傳送**高**]、 [**低**] 或 [**內文**的優先順序。

<sup>*</sup>這些屬性只能中可用增強摘要報告。

### <a name="extended-reports"></a>延伸的報告

郵件追蹤開頭 [ **Downloadable 報告**] 區段中有可用 （已完成） 的延伸報告。幾乎所有的增強摘要報告是資訊的可在 （除了**origin_timestamp**和**delivery_priority**） Extended 報表。延伸報表中只有下列額外的資訊：

- **client_ip**： 電子郵件伺服器或送出郵件的郵件用戶端的 IP 位址。

- **client_hostname**： 的主機名稱或電子郵件伺服器或送出郵件的郵件用戶端的 FQDN。

- **server_ip**： 來源或目的地伺服器的 IP 位址。

- **server_hostname**： 的主機名稱或目的地伺服器的 FQDN。

- **source_context**： 與**來源**欄位相關的額外資訊。例如：

   - `Protocol Filter Agent`

   - `3489061114359050000`

- **來源**： 負責事件的 Exchange Online 元件。例如：

   - `AGENT`

   - `MAILBOXRULE`

   - `SMTP`

- **event_id**： 這些對應至所述來[尋找此郵件的相關的記錄](#find-related-records-for-this-message)] 區段中的**郵件事件**值。

- **internal_message_id**： 指派 Exchange Online 伺服器目前正在處理之郵件的郵件識別碼。

- **recipient_address**： 郵件的收件者的電子郵件地址。以分號 （;） 分隔多個電子郵件地址。

- **recipient_count**： 收件者的郵件總數。

- **related_recipient_address**： 搭配`EXPAND`、 `REDIRECT`，及`RESOLVE`事件來顯示其他收件者電子郵件相關聯之郵件的地址。

- **參考**： 此欄位會包含特定事件類型的其他資訊。例如：

   - **DSN**： 包含相關聯的傳遞狀態通知 （又稱為 DSN、 未傳遞回報、 NDR 或彈跳訊息） **message_id**值，則使用 DSN 產生此事件之後的報告] 連結。如果這是 DSN 郵件，此欄位會包含 DSN 產生的原始郵件的**message_id**值。

   - **展開**： 含有相關郵件的**related_recipient_address**值。

   - **接收**： 如果郵件產生供其他程序 （例如 [收件匣規則） 可能會包含相關訊息**message_id**值。

   - **傳送**： 含有任何 DSN 郵件的**internal_message_id**值。

   - **傳輸**： 包含**internal_message_id**值被分叉時 （例如，依內容轉換、 郵件收件者限制或代理程式） 的郵件。

   - **MAILBOXRULE**： 包含**internal_message_id**值造成產生外寄郵件的收件匣規則的內送郵件。

   其他類型的事件，此欄位是通常是空白的。

- **return_path**： 指定傳送的郵件**MAIL FROM**命令所傳回的電子郵件地址。雖然此欄位永遠不是空白的它可以具有 null 寄件者地址值以表示`<>`。

- **message_info**： 郵件的其他資訊。例如：

   - 郵件起始日期-時間的 utc`DELIVER`和`SEND`事件。起始日期-時間是當郵件先輸入 Exchange Online 組織的時間。UTC 的日期-時間表示以 ISO 8601 日期時間格式： `yyyy-mm-ddThh:mm:ss.fffZ`，其中`yyyy`= 年`mm`= 月、 `dd` = 一天`T`表示開頭的時間的元件， `hh` = 小時、 `mm` = 分鐘、 `ss` = 第二、 `fff`= 分數的第二、 和`Z`代表`Zulu`，這是另一種方式皆 UTC。

   - 驗證錯誤。例如，您可能會看到值`11a`和驗證錯誤發生時所使用的驗證類型。

- **tenant_id**： 代表 Exchange Online 組織的 GUID 值 (例如`39238e87-b5ab-4ef6-a559-af54c6b07b42`)。

- **original_server_ip**： 原始伺服器的 IP 位址。

- **custom_data**： 包含與特定事件類型相關的資料。如需詳細資訊，請參閱下列各節。

#### <a name="customdata-values"></a>custom_data 值

**Custom_data**欄位`AGENTINFO`事件由 Exchange Online 的代理程式的各種可用來記錄訊息處理的詳細資訊。下列各節說明一些更有趣但代理程式。

#### <a name="spam-filter-agent"></a>垃圾郵件篩選器代理程式

開頭的**custom_data**值`S:SFA`是來自垃圾郵件篩選器代理程式。下表說明重要詳細資料：

|**值**|**描述**|
|:-----|:-----|
|`SFV=NSPM`|郵件標記為非垃圾郵件，並傳送給預定的收件者。|
|`SFV=SPM`|內容篩選已將郵件標記為垃圾郵件。|
|`SFV=BLK`|已略過篩選，且郵件來自封鎖的寄件者，所以封鎖郵件。|
|`SFV=SKS`|內容篩選在處理郵件前，已將郵件標記為垃圾郵件。這包括符合傳輸規則因此自動標記為垃圾郵件，因而略過所有其他篩選的郵件。|
|`SCL=<number>`|如需不同 SCL 值和其意義的詳細資訊，請參閱[Spam confidence levels](https://technet.microsoft.com/library/jj200686.aspx)。|
|`PCL=<number>`|網路釣魚信賴等級 (PCL) 值的郵件。這些可以解譯一樣 SCL[垃圾郵件信賴等級](https://technet.microsoft.com/library/jj200686.aspx)中記載的值。|
|`DI=SB`|已封鎖郵件的寄件者。|
|`DI=SQ`|已隔離郵件。|
|`DI=SD`|已刪除郵件。|
|`DI=SJ`|郵件已傳送給收件者的垃圾郵件] 資料夾。|
|`DI=SN`|郵件已透過較高風險傳遞集區。如需詳細資訊，請參閱 ＜[高風險傳遞集區的外寄郵件](https://technet.microsoft.com/library/jj200746.aspx)。|
|`DI=SO`|已透過標準輸出傳遞集區路由傳送郵件。|
|' SFS = []|SFS = [b] '|這表示已符合垃圾郵件規則。|
|`IPV=CAL`|因為 IP 位址指定於連線篩選的 [IP 允許] 清單中，所以已透過垃圾郵件篩選允許郵件。|
|`H=<EHLOstring>`|連線的電子郵件伺服器的 HELO 或 EHLO 字串。|
|`PTR=<ReverseDNS>`|傳送 IP 位址 (也稱為反向 DNS 位址) 的 PTR 記錄。|

針對類似的垃圾郵件篩選過的訊息範例**custom_data**值：

`S:SFA=SUM|SFV=SPM|IPV=CAL|SRV=BULK|SFS=470454002|SFS=349001|SCL=9|SCORE=-1|LIST=0|DI=SN|RD=ftmail.inc.com|H=ftmail.inc.com|CIP=98.129.140.74|SFP=1501|ASF=1|CTRY=US|CLTCTRY=|LANG=en|LAT=287|LAT=260|LAT=18;`

#### <a name="malware-filter-agent"></a>惡意程式碼篩選器代理程式

開頭的**custom_data**值`S:AMA`來自惡意軟體篩選器代理程式。下表說明重要詳細資料：

|**值**|**描述**|
|:-----|:-----|
|' AMA = SUM|v = 1|` or `AMA = EV|v = 1'|郵件已決定要包含惡意程式碼。`SUM`指出惡意程式碼可能已偵測到的任意數量的引擎。`EV`會指出特定引擎所偵測到惡意程式碼。當惡意程式碼會偵測引擎這會觸發後續動作。|
|`Action=r`|已取代郵件。|
|`Action=p`|已略過郵件。|
|`Action=d`|已延遲郵件。|
|`Action=s`|已刪除郵件。|
|`Action=st`|已略過郵件。|
|`Action=sy`|已略過郵件。|
|`Action=ni`|已拒絕郵件。|
|`Action=ne`|已拒絕郵件。|
|`Action=b`|已封鎖郵件。|
|`Name=<malware>`|偵測到之惡意程式碼的名稱。|
|`File=<filename>`|含有惡意程式碼之檔案的名稱。|

郵件包含惡意程式碼範例**custom_data**值看起來如下：

`S:AMA=SUM|v=1|action=b|error=|atch=1;S:AMA=EV|engine=M|v=1|sig=1.155.974.0|name=DOS/Test_File|file=filename;S:AMA=EV|engine=A|v=1|sig=201707282038|name=Test_File|file=filename`

#### <a name="transport-rule-agent"></a>傳輸規則代理程式

開頭的**custom_data**值`S:TRA`是來自傳輸規則代理程式郵件流程規則 （也稱為傳輸規則）。下表說明重要詳細資料：

|**值**|**描述**|
|:-----|:-----|
|' ETR|ruleId =<guid>`|已符合的規則 ID。|
|`St=<datetime>`|日期和時間 UTC 規則比對發生時。|
|`Action=<ActionDefinition>`|所套用的動作。如需可用動作的清單，請參閱[Mail flow 規則動作在 Exchange Online](https://technet.microsoft.com/library/jj919237.aspx)。|
|`Mode=<Mode>`|規則的模式。有效值如下：<br/>• **Enforce**： 將強制執行規則上的所有動作。 <br/>•**搭配原則提示來測試：**： 將會傳送任何 「 原則提示 」 動作，但在應對不到其他強制執行動作。 <br/>• [**不搭配原則提示的測試**： 動作將會列在記錄檔中，但不是會通知寄件者以任何方法，並在應對不到強制執行動作。|

郵件流程規則的條件會比對範例**custom_data**值的郵件看起來如下：

`S:TRA=ETR|ruleId=19a25eb2-3e43-4896-ad9e-47b6c359779d|st=7/17/2017 12:31:25 AM|action=ApplyHtmlDisclaimer|sev=1|mode=Enforce`
