# <a name="session-policies-in-office-365-cloud-app-security"></a>Office 365 雲端 App 安全性中的工作階段原則

|評估 * *\>**|規劃 * *\>**|部署 * *\>**|使用率 * * *|
|:-----|:-----|:-----|:-----|
|[啟動評估](office-365-cas-overview.md) <br/> |[開始規劃](get-ready-for-office-365-cas.md) <br/> |您在此處 ！  <br/> [下一步](ocas-access-policies.md) <br/> |[開始使用](utilization-activities-for-ocas.md) <br/> |

Office 365 雲端 App 安全性工作階段原則啟用即時工作階段層級監視，並細微的可視性 Office 365 應用程式所能採取不同的動作為使用者工作階段的原則。 而不是允許或封鎖存取完全，工作階段控制您可以同時監視使用條件式存取應用程式控制項的反向 proxy 功能的工作階段及/或限制特定的工作階段活動允許存取。

例如，您可以決定，從受管理的裝置，或來自特定位置工作階段，您想要允許使用者存取應用程式，但也限制的機密檔案下載，或需要特定文件受到保護時下載。 工作階段原則可讓您設定這些使用者工作階段的控制項，並允許存取，並可讓您：

- [監視所有活動](#monitor-all-activities)

- [封鎖所有的下載項目](#block-all-downloads)

- [封鎖特定活動](#block-specific-activities)

- [保護上下載的檔案](#protect-files-on-download)

## <a name="prerequisites-to-using-session-policies"></a>若要使用的工作階段原則的必要條件

- Azure AD Premium P1 授權

- 相關的 Office 365 應用程式應該是 [使用條件式存取應用程式控制部署](ocas-deploy-conditional-access-app-control.md)

-  [Azure AD 條件式存取原則](https://docs.microsoft.com/azure/active-directory/active-directory-conditional-access-azure-portal) 應該在將使用者重新導向至 Office 365 雲端 App 安全性的地方

## <a name="create-an-azure-ad-conditional-access-policy"></a>建立 Azure AD 條件式存取原則

Azure Active Directory 條件式存取原則與 Cloud App Security 工作階段原則適用於搭配檢查每個使用者工作階段，並決定每個應用程式的原則。 若要在 Azure AD 中設定的條件式存取原則，請遵循此程序：

1. 設定 [Azure AD 條件式存取原則](https://docs.microsoft.com/azure/active-directory/active-directory-conditional-access-azure-portal) 與使用者或群組的使用者與您想要與條件式存取應用程式控制項的控制項的應用程式的工作分派。 附註： 已 [部署與條件式存取應用程式控制項](ocas-deploy-conditional-access-app-control.md)僅應用程式 將會受到此原則。

2. 將使用者路由傳送至 Office 365 雲端 App 安全性藉由選取 [ **使用條件式存取應用程式控制強制執行限制**  **工作階段**中 ] 頁面。

## <a name="create-a-cloud-app-security-session-policy"></a>建立雲端 App 安全性工作階段原則

若要建立新的工作階段原則，請遵循此程序：

1. 在 [入口網站中，選取 **控制項** 後面接著 **原則**。

2. 在 [ **原則** ] 頁面上，按一下 [ **建立原則** 並選取 [ **工作階段原則**。

3. 在 [ **工作階段原則** ] 視窗中，指派原則的名稱

4. 在 **工作階段的控制項類型** 欄位：
    
    - 選取 [ **僅限監視器** 如果您只想要監視使用者活動。 此選取項目將會建立您所選取其中所有登入、 啟發式下載及活動類型將都下載的應用程式的監視器唯一原則。
    
    - 選取 [ **控制 （使用 DLP 時) 的檔案下載** 如果您想要監視使用者活動。 您可以採取其他動作，例如封鎖或保護使用者的下載項目。
    
    - 選取 [ **封鎖活動** 來封鎖特定的活動，您可以選擇使用 **活動類型** 篩選器。 從選取的應用程式的所有活動會是監控 （和報告在活動記錄）。 如果您選取 [ **區塊**，將會封鎖您選取的特定活動 巨集指令。 如果您選取 [ **測試**您所選取的特定活動會引發警示 巨集指令與已開啟的提醒。

5. [ **活動來源** 中 **比對下列所有的活動** ] 區段中，選取要套用至該原則的其他活動篩選。 這些篩選器可以包含下列選項：
    
    - **裝置標記**： 使用此篩選器來識別未受管理的裝置。
    
    - **位置**： 使用此篩選器來識別不明 （並因此風險） 的位置。
    
    - **IP 位址**： 使用此篩選條件來篩選每個 IP 位址] 或 [使用先前指派的 IP 位址標記。
    
    - **使用者代理程式標記**： 使用此篩選器啟用的啟發學習法來識別行動裝置和傳統型應用程式。 此篩選器可以設定為等於或不等於 **原生用戶端**。 此篩選應該比照您每個雲端應用程式的行動裝置和傳統型應用程式。<br>附註： 工作階段原則不支援行動裝置和傳統型應用程式。 桌面應用程式和行動應用程式可以也封鎖或允許藉由建立存取原則。

6. 如果您選取選項來 **控制檔案下載 （使用 DLP 時)**，請在 **活動來源**] 下  **符合下列所有檔案**中 ] 區段中，選取要套用至該原則的其他檔案篩選。 這些篩選器可以包含下列選項：
        
    - **分類標籤** -如果您的組織使用 Azure 資訊保護和其分類標籤所受保護您的資料後，請使用此篩選。 您可以篩選套用至他們的分類標籤為基礎的檔案。 如需有關整合使用 Azure 資訊保護，請參閱 [Azure 資訊保護整合](https://docs.microsoft.com/cloud-app-security/azip-integration)。
        
    - **檔案名稱** -使用此篩選器原則套用至特定的檔案。
        
    - **檔案類型** -使用此篩選器原則套用至特定檔案類型，例如封鎖下載所有.xls 檔案。
    
    - 在 [ **內容檢查** ] 區段中，設定是否要讓 DLP 引擎掃描的文件和檔案內容。
    
    - 在 [ **動作**] 下選取下列其中一個下列項目：
        
        - **測試 （監視所有活動）**： 設定此巨集指令，明確允許下載根據您所設定的原則篩選器。
        
        - **區塊 （封鎖檔案下載和監視所有活動）**： 設定明確封鎖根據您所設定的原則篩選器下載此巨集指令。 如需詳細資訊，請參閱 [封鎖下載運作的方式](https://docs.microsoft.com/en-us/cloud-app-security/session-policy-aad#block-download)。
        
        - **Protect （套用分類標籤來下載及監視所有活動）**： 此選項才可用如果您選取了 **控制檔案下載 （與 DLP)**  **工作階段原則**] 底下。 如果您的組織使用 Azure 資訊保護，您可以設定 **巨集指令** 若要套用的分類標籤設定 Azure 資訊保護的檔案。 如需詳細資訊，請參閱 <<c0>如何保護下載運作。

7. 您可以 **建立的原則嚴重性與每個比對事件警示** 和設定的警示限制。 選取是否要為一封電子郵件、 文字訊息，或兩者警示。

## <a name="monitor-all-activities"></a>監視所有活動

當您建立工作階段原則時，原則會比對每個使用者工作階段會重新導向至工作階段控制，而不是應用程式直接。 使用者會看到監視的通知，以讓他們知道其工作階段會受監視。

如果您不想要通知使用者他們正在受監視，您可以停用的通知訊息。

1. 在設定商旅中，選取 [ **一般設定**]。

2. 然後，在 **條件式存取應用程式控制** 選取 **使用者監視** 並取消選取 **通知使用者** ] 核取方塊。

若要保留內的工作階段、 條件式存取應用程式控制取代所有相關的 Url、 Java 指令碼和 cookie 的使用者，與 Office 365 雲端 App 安全性 Url 的應用程式工作階段內。 例如，如果應用程式會傳回有連結的頁面，其網域結束於`myapp.com`，條件式存取應用程式控制會取代以類似的網域中的連結`myapp.com.us.cas.ms`。 這種方式在整個工作階段的監視可以 Office 365 雲端 App 安全性。

條件式存取應用程式控制記錄每個使用者工作階段，透過它路由傳送的流量記錄檔。 流量記錄檔包括時間、 IP、 使用者代理程式，瀏覽的 Url，並上傳及下載的位元組數。 這些記錄檔會經過分析，連續的報表， **雲端 App 安全性條件式存取應用程式控制項**，會新增至雲端探索儀表板中的雲端探索報表的清單。

### <a name="to-export-these-logs"></a>若要匯出這些記錄檔：

1. 前往 [設定商旅，然後按一下 [ **條件式存取應用程式控制**。

2. 在表格的右側，按一下 [匯出] 按鈕。<br>![匯出] 按鈕](media/image3.png)<br>

3. 選取報表的範圍，然後按一下 [ **匯出**]。 此程序可能需要一些時間。

### <a name="to-download-the-exported-log"></a>若要下載匯出的記錄檔：

1. 準備報表之後，請移至 [ **設定** ，然後 **匯出報告**。

2. 在表格中，請從 **條件式存取應用程式控制項的流量記錄檔**的清單中選取相關的報告 並按一下 [下載]。<br>![下載] 按鈕](media/image4.png)<br>

## <a name="block-all-downloads"></a>封鎖所有的下載項目

當 **封鎖** 設為 **巨集指令** 您想要取得雲端 App 安全性工作階段原則中，條件式存取應用程式控制可防止使用者下載每個原則的檔案篩選器的檔案。 下載事件辨識 Office 365 雲端 App 安全性每個應用程式當使用者開始下載。 條件式存取應用程式控制介入可防止執行的即時為止。 條件式存取應用程式控制使用者已經啟動下載收到接收的訊號時，傳回 **下載限制** 訊息給使用者，並以文字檔案取代下載的檔案。 可設定及透過 「 工作階段原則來自訂文字檔的訊息給使用者。

## <a name="block-specific-activities"></a>封鎖特定活動

時 **封鎖活動** 設為 **活動類型**中，您可以選取要封鎖特定的應用程式中的特定活動。 將監視從選取的應用程式的所有活動，並將其活動記錄檔中報告。 如果您選取 [ **區塊**，將會封鎖您選取的特定活動 巨集指令。 如果您選取的 **測試**巨集指令，而且具有提醒您所選取的特定活動會引發警示開啟。

**封鎖特定活動** 並將其套用至特定群組，以建立您的組織使用完整唯讀模式。

## <a name="protect-files-on-download"></a>保護上下載的檔案

選取 [ **封鎖活動** 來封鎖特定的活動，您可以找到使用 **活動類型** 篩選器。 從選取的應用程式的所有活動會是監控 （和報告在活動記錄）。 如果您選取 [ **區塊**，將會封鎖您選取的特定活動 巨集指令。 如果您選取 [ **測試**您所選取的特定活動會引發警示 巨集指令與已開啟的提醒。

當 **保護** 設為 **巨集指令** 所要採取的 Cloud App Security 工作階段原則中，條件式存取應用程式控制強制執行原則的檔案篩選器每一個檔案的標籤和後續保護。 標籤已在 [Azure 資訊保護主控台和 **保護** 必須選取內就會出現在 Cloud App Security 原則選項的標籤。 當選取標籤，並下載檔案後，符合準則的 Cloud App Security 原則時，label 及對應的保護 （具有權限） 會套用至在下載時的檔案。 原始的檔案會保持為-是在雲端應用程式中，而現在保護下載的檔案。 嘗試存取檔案的使用者必須符合取決於保護套用的權限需求。
