# <a name="create-and-apply-information-management-policies"></a>建立與套用資料的資訊管理原則

資訊管理原則啟用多久要保留稽核人員用來做什麼內容，並新增條碼的內容控制項為組織或文件的標籤。原則可協助強制遵守法律及政府規定或內部商務程序。身為管理員，您可以設定原則以控制如何追蹤文件及多久要保留的文件。
  
您可以建立資訊管理原則可以在三個不同的位置的網站階層中從最大到最窄：
  
- 建立網站集合內的多個內容類型上使用的原則。
    
- 建立網站內容類型的原則。
    
- 建立清單或文件庫的原則。
    
如需詳細資訊，請參閱[資訊管理原則簡介](intro-to-info-mgmt-policies.md)。
  
## <a name="create-a-policy-for-multiple-content-types-within-a-site-collection"></a>建立網站集合內的多個內容類型的原則
<a name="__toc261001590"> </a>

為確保資訊原則套用至網站集合內的特定類型的所有文件，請考慮建立網站集合層級的原則與稍後將原則套用至內容類型。這些被所謂的網站集合原則。 
  
1. 在網站集合首頁\>**設定**![標題列上的 SharePoint 2016 設定] 按鈕。](media/1c22d2d8-39e0-4930-82c6-c3eee44211d3.png) \> **網站設定**。
    
    在 SharePoint 群組連接網站中，按一下 [**設定**]、 按一下 [**網站內容**]，和 [**網站設定**。 
    
2. 在 [網站設定] 頁面上的 [**網站集合管理** \> **內容類型原則範本**。 
  
![在 [網站設定] 頁面上的內容類型原則範本連結](media/26d3466a-23ec-443f-88f0-2aaff38e992b.png)
  
3. 在 [原則] 頁面上\>**建立**。 
    
4. 輸入的名稱和描述的原則，並接著撰寫說明使用者原則是針對簡短的原則陳述式。
    
5. 請參閱下一個區段上建立網站內容類型以了解如何設定您想要將原則建立關聯之的功能的原則。 
    
6. 選擇 [確定]。

## <a name="create-a-policy-for-a-list-library-or-folder-location-based-retention-policy"></a>建立清單、文件庫或資料夾的原則 (位置保留原則)
<a name="__create_a_policy"> </a>

您可以定義僅適用於特定清單、 文件庫或資料夾的保留原則。不過，如果您建立保留原則如此一來，您不能重複使用此原則在其他清單、 文件庫、 資料夾或網站與您無法將網站集合原則套用至基礎的位置原則。
  
如果您想要將單一的保留原則套用至所有類型的單一位置中的內容，您很有可能要使用以位置為主的保留。中其他大多數的情況下，您會想要確認已指定的所有內容類型的保留原則。
  
 除非您選擇要中斷繼承和定義新的保留原則層級的子每一個子資料夾會繼承其父項、 保留原則。 
  
如果您想要定義保留以外的資訊管理原則至清單或文件庫，您需要定義與清單或文件庫相關聯的每個個別清單內容類型的資訊管理原則。
  
 如果在任何時候您決定切換從內容類型至清單或文件庫的位置型原則、 保留原則將用作位置為基礎的原則。所有其他管理原則 （稽核、 條碼，以及條碼） 將會繼承自相關的內容類型。 
  
 架構的位置原則可用來停用的文件庫和資料夾以保留功能停用網站集合之用。這可讓網站集合管理員以確保其內容類型原則清單中系統管理員的架構的位置原則不會被覆寫。 
  
您至少需要管理清單權限變更清單或文件庫的資訊管理原則設定。
  
1. 瀏覽至想要指定的資訊管理原則的清單或文件庫。 
    
2. 在功能區上選擇 [**文件庫**或**清單**] 索引標籤\>**文件庫設定**] 或 [**清單設定**。
    
    在 SharePoint Online 中，按一下 [**設定**] 並再按一下 [**清單設定**] 或 [**文件庫設定**。 
    
3. [**權限與管理**\> **資訊管理原則設定**。
  
![文件庫設定] 頁面上的資訊管理原則連結](media/9fa6d366-6aab-49e1-a05c-898ac6f536e6.png)
  
4. 在 [資訊管理原則設定] 頁面上，確認的清單或文件庫的保留來源設為文件庫和資料夾。 
  
如果**內容類型**顯示為來源，請按一下 [**變更來源**] 和 [**文件庫和資料夾**。您會收到通知內容類型的保留原則將會被忽略。選擇 **[確定]**。 
    
5. 在 [編輯原則] 頁面上的 [**文件庫以保留排程**] 下輸入您要建立之原則的簡短描述。 
    
6. 選擇 [**新增保留階段...**
    
     請注意下記錄，您可以選擇選取 [記錄] 選項的定義不同保留階段定義記錄的保留原則。 
    
7. 在階段 [屬性] 對話方塊中，選取 [可指定當設定到期的文件或項目保留期間] 選項。執行下列其中一項：
    
  - 若要設定之下**事件**的日期屬性為基礎的到期日期\>**這個階段基礎關閉項目上的 [日期] 屬性**，然後選取 [文件或項目動作 （例如，建立或修改） 與此動作 （之後的時間單位例如的天數、 月數或年數） 當您想要到期的項目。 
    
  - 若要使用的自訂保留公式來決定到期日，選擇 [**這個伺服器上安裝自訂保留公式來設定**。 
    
    > [!NOTE]
    >  如果您的系統管理員所自訂公式設定，這個選項只有。 
  
  - 在 [**巨集指令**，指定您想要的文件或項目到期時。若要啟用特定動作至文件或項目 （例如刪除） 的發生，請從清單中選取動作。 
    
8. 只有在您定義的原則清單、 文件庫或內容類型已有與其相關聯的工作流程使用**啟動工作流程**] 選項。您再將獲得選擇所要選擇的工作流程。 
    
9. [**循環**，選擇 [**重複這個階段動作...** 並輸入想要再度發生的動作的頻率。 
    
    > [!NOTE]
    >  只有當您所選取的巨集指令可以重複使用此選項。例如，您無法設定**永久刪除**動作的循環。 
  
10. 選擇 [確定]。
    
## <a name="apply-a-site-collection-policy-to-a-content-type"></a>將網站集合原則套用至內容類型
<a name="__apply_a_site"> </a>

如果資訊管理原則已經已為網站建立為網站集合原則，您可以將其中一個原則套用至內容類型。依照此您可以將相同的原則套用至多種內容類型的網站集合中不共用相同的上層內容類型。
  
 若您想要將原則套用至網站集合中的多個內容類型和您必須設定受管理的中繼資料服務，您可以使用內容類型發佈發佈出資訊管理原則至多個網站集合。請參閱一節[將原則套用到整個網站集合的內容類型](create-info-mgmt-policies.md#__apply_a_policy)的詳細資訊。 
  
1. 瀏覽至清單或文件庫包含您要套用原則的內容類型。
    
2. 在功能區上選擇 [**文件庫**或**清單**] 索引標籤\>**文件庫設定**] 或 [**清單設定**。
    
    在 SharePoint Online 中，按一下 [**設定**] 並再按一下 [**清單設定**] 或 [**文件庫設定**。 
    
3. [**權限與管理** \> **資訊管理原則設定**。
  
![文件庫設定] 頁面上的資訊管理原則連結](media/9fa6d366-6aab-49e1-a05c-898ac6f536e6.png)
  
4. 確認原則來源已設定與**內容類型**，並在 [**內容類型原則**選取您要套用之原則的內容類型。 
    
5. 在 [**指定原則** \> **使用網站集合原則**]，然後選取 [您想要從清單中套用的原則。 
    
    > [!NOTE]
    >  如果未**使用網站集合原則**選項，沒有網站集合原則已定義的網站集合。 
  
6. 選擇 [確定]。
    
     如果您正在處理的文件庫或清單支援多種內容類型的管理、**內容類型**下您可以選擇您要指定的資訊管理原則的內容類型。這將會前往直接上面的步驟 5。 
    
## <a name="apply-a-policy-across-site-collections"></a>將原則套用到網站集合

## <a name="create-a-template-from-an-existing-policy-to-use-across-site-collections"></a>從現有的原則以使用跨網站集合中建立範本
<a name="__toc262125409"> </a>

您可以定義資訊管理原則，然後建立從其使用跨多個網站集合所需的 [範本。如果您想要有資訊原則、 的備份或使用內容類型發佈的跨網站集合套用某個原則之替代方法以可用也可以使用此方法。您可以從一個網站集合匯出原則，然後匯入它已儲存的位置或另一個網站集合中建立範本或之原則的備份。
  
> [!IMPORTANT]
>  如果您使用匯出/匯入功能的方式，讓一組的原則範本，請記住，在於原則.xml 檔案中的唯一識別碼。因此，您不能匯入該原則至網站一次以上而不變更此唯一識別碼。 
  
### <a name="export-a-policy"></a>匯出原則
<a name="__toc260646790"> </a>

1. 在 [網站集合首頁] 頁面上選擇 [**設定**![執行的網站設定的小型設定齒輪表示。](media/a47a06c3-83fb-46b2-9c52-d1bad63e3e60.png) \> **網站設定**。
    
    在 SharePoint 群組連接網站中，按一下 [**設定**]、 按一下 [**網站內容**]，和 [**網站設定**。 
    
2. 在 [網站設定] 頁面上的 [**網站集合管理** \> **內容類型原則範本**。 
  
![在 [網站設定] 頁面上的內容類型原則範本連結](media/26d3466a-23ec-443f-88f0-2aaff38e992b.png)
  
3. 選擇您想要匯出的原則\>捲軸底部\>**匯出**。
    
4. 提示儲存或開啟的檔案，並選擇 [**儲存**，然後選取要儲存之檔案的位置。請務必選取 [可匯入之原則的網站集合的位置。
    
5. [下載完成] 對話方塊顯示時，選擇 [**關閉**]。
    
### <a name="import-a-policy-to-a-different-site-collection"></a>原則匯入至不同的網站集合
<a name="__toc260646791"> </a>

匯入的資訊管理原則可讓您可以套用至網站或清單層級任何指定的網站集合內的多個內容類型。執行此動作的優點有兩個層面： 您不需要重新定義及套用該原則上每個內容類型，以及您可以更輕鬆地管理原則修改所進行的變更只是一個位置原則。
  
1. 在您要套用之原則的網站集合的 [首頁] 頁面上選擇 [**設定**![執行的網站設定的小型設定齒輪表示。](media/a47a06c3-83fb-46b2-9c52-d1bad63e3e60.png) \> **網站設定**。
    
    在 SharePoint 群組連接網站中，按一下 [**設定**]、 按一下 [**網站內容**]，和 [**網站設定**。 
    
2. 在 [網站設定] 頁面上的 [**網站集合管理** \> **內容類型原則範本**。
    
3. 在 [原則] 頁面上\>**匯入** \> **瀏覽]** 尋找原則的 XML 檔案。 
    
4. 選取已在其中儲存之原則的 XML 檔案\>**開啟**。 
    
5. 在匯入網站集合原則] 頁面上\>**匯入**新增原則至網站集合。 
    
匯入的原則現在可以套用至一或多個網站或清單層級的內容類型。
