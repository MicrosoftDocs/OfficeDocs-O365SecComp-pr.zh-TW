# <a name="review-conversations-in-advanced-ediscovery"></a>檢閱在進階電子文件中的交談 

立即訊息是提問、 分享想法，或跨大型對象快速通訊便利方式。 為立即訊息平台，例如 Microsoft Teams，成為企業共同作業的核心，組織必須評估其的 eDiscovery 工作流程如何解決這些新的表單的通訊和共同作業。 

進階電子文件中的交談重建功能被設計來協助您識別內容相關的內容，並產生不同的交談檢視。 這項功能可讓您有效地和快速檢閱完整的立即訊息對話 （也稱為*執行緒模式下執行交談*） 而產生的平台像是 Microsoft Teams。

與交談重建，您可以使用內建功能來重建、 進行檢閱，以及匯出執行緒的交談。 使用進階電子文件交談重建來：

- 保留唯一郵件層級中繼資料內交談的所有郵件。

- 收集周圍您的搜尋結果的內容相關訊息。

- 檢閱、 加上註解，並 redact 執行緒的交談。

- 匯出個別郵件或執行緒的交談

## <a name="terminology"></a>術語

以下是一些定義，以協助您開始使用交談重建。

- **郵件：** 代表交談的最小單位。 在大小、 結構和中繼資料，郵件可能會有所不同。 

- **交談：** 代表一個或多封郵件的群組。 跨不同的應用程式，可能會不同的方式表示交談。 某些應用程式，在沒有明確的動作所產生的回覆至現有的郵件。 交談是明確地形成，由於此使用者動作。 例如，以下是在 Microsoft Teams 的通道交談的螢幕擷取畫面。

   ![Microsoft Teams 通道交談](../media/threadedchat.png)

   在其他應用程式 （例如 teams 1xN 聊天訊息），並沒有正式回覆鏈結，改為郵件顯示為 「 平面河的郵件 」 的單一執行緒內。 在這些類型的應用程式，從一群發生在特定時間內的郵件會推斷交談。 這個 「 虛群組 」 的郵件 （而非回覆鏈結） 代表 「 往返 」 交談相關的特定主題感興趣。 

## <a name="step-1-run-a-search"></a>步驟 1： 執行搜尋

識別相關 custodians 和內容位置之後，您可以建立搜尋，以尋找潛在相關的內容。 進階電子文件探索案例中的 [**搜尋**] 索引標籤，您可以按一下 [**新增搜尋**，並遵循精靈建立搜尋。 如需如何建立搜尋，建置搜尋查詢，和檢視搜尋結果，請參閱[收集資料的大小寫](create-search-to-collect-data.md)。

## <a name="step-2-create-a-conversation-review-set"></a>步驟 2： 建立交談檢閱設定

檢閱集中，您可以搜尋、 標記、 加上註解，以及 redact 文件、 電子郵件訊息和聊天交談。 在 [進階電子文件，您可以自訂交談、 根據中的個別郵件或執行緒的交談的檢閱。 這取決於檢閱集的類型，您將新增至步驟 1 中建立的搜尋結果。 有兩種不同檢閱設定： 
  
  - **標準檢閱設定：** 在交談中的郵件會處理，並顯示為個別項目。 
  
  -  **交談檢閱設定：** 交談中的郵件會個別處理，但顯示在 [交談] 檢視中。 交談中檢閱設定，您可以加上註解、 標記，redact 執行緒的交談] 檢視中的郵件。 

如需如何檢閱，以及管理內容中檢閱設定，請參閱[管理檢閱設定](managing-review-sets.md)。 

## <a name="step-3-enable-conversation-retrieval-options"></a>步驟 3： 啟用交談擷取選項

您已檢閱並完成您的搜尋查詢之後，您可以檢閱組新增搜尋結果。 當您新增您的搜尋結果成檢閱集時，原始的資料會複製到 Azure 儲存體區域，以利檢閱和分析處理程序。 如需有關將搜尋結果新增至檢閱設定，請參閱[新增搜尋結果，以檢閱設定](add-data-to-review-set.md)。 

當您從交談中新增資料，檢閱設定時，您可以使用交談擷取選項擴大搜尋範圍，並將包含內容相關訊息。 設定交談擷取選項之後，可以發生下列情形：

  ![交談擷取](../media/messagesandconversations.png)
  
1. 使用關鍵字和日期範圍查詢，搜尋所傳回落點的*訊息 3*。 此郵件為較大的交談， *CRC1*所說明的一部分。 
  
2. 當您新增資料至檢閱設定並啟用交談擷取選項時，將會回復進階電子文件，並收集*CRC1*中的其他項目。 
  
3. 項目已新增至檢閱設定之後，您可以檢閱來自*CRC1*的所有個別郵件。 



若要啟用交談擷取：
  
1. 進階電子文件探索案例的 [**搜尋**] 索引標籤上選擇 [搜尋]，，然後按一下彈出式頁面上的 [**新增]，檢閱設定**。
  
2. 選取現有的檢閱設定或建立檢閱設定。 新增搜尋結果，以 standard 或交談檢閱設定時，您可以設定擷取選項。
  
3. 設定您想要在搜尋中，展開，然後按一下 [**新增**] 以開始程序的內容來源的交談擷取選項。  
  
4. 在 [**工作**] 索引標籤上的 [**新增]，檢閱設定**工作已完成之後，您可以開始檢閱交談。

## <a name="step-4-review-conversations-in-the-review-set"></a>步驟 4： 檢閱檢閱組中的交談

已處理並新增至檢閱組內容之後，您可以開始檢閱檢閱集中的資料。 檢閱功能會根據是否內容已新增至標準檢閱組或交談檢閱集不同。 

### <a name="reviewing-conversations-in-a-standard-review-set"></a>檢閱交談標準中的檢閱設定

標準中檢閱設定，郵件會處理並顯示為個別項目，類似於如何依據儲存在信箱資料夾。 在此工作流程，做為個別的項目處理每個訊息。 因此，執行緒的摘要和匯出選項無法使用標準檢閱集合中。 

### <a name="reviewing-conversations-in-a-conversation-review-set"></a>檢閱在交談檢閱交談設定

交談中檢閱設定，個別的郵件都會一起執行緒，並當作交談來呈現。 這可讓您檢視及匯出內容相關的交談。 下列各節說明檢閱和匯出交談檢閱集合中的交談。

#### <a name="reviewing-conversations"></a>檢閱交談

在交談中檢閱設定，您可以使用下列選項以利檢閱程序。

- **依交談群組：** 群組內相同對話訊息一起可協助簡化並加速其檢閱程序的使用者。 

- **摘要檢視：** 會顯示執行緒的交談。 在此檢視中，您可以看見整個交談，並也存取每個個別訊息的中繼資料。  
  
   - 檢視個別郵件的中繼資料
   
   - 下載個別郵件

- **文字檢視：** 提供整個交談的擷取的文字。 

- **加上註解檢視：** 可讓您標記的對話執行緒檢視。 交談中的所有郵件會都共用相同的註解文件。

- **標記：** 檢視時交談中檢閱設定，您可以檢視及套用標籤的 [**編碼] 面板**。

- **重新執行交談轉換：** 當郵件新增至交談檢閱設定時，轉換執行作業時自動建立執行緒的摘要，並加上註解的檢視。 如果交談重建作業失敗時，您可以檢閱集合中的 [**巨集指令 > 建立交談 Pdf**重新執行轉換工作。


#### <a name="exporting-conversations"></a>匯出的交談

在交談中檢閱設定，您可以設定下列選項來匯出交談：

![匯出](../media/export.png)

a. 中繼資料] 選項

   - **載入檔案：** 包含每個個別訊息、 電子郵件和文件中繼資料。 沒有對話中的每一封郵件的一列。 

   - **標記：** 從您檢閱程序的標記會包含在中繼資料檔案。 交談中的郵件會共用相同的標籤。 

b. 對話選項
  
   - **交談檔案：** 當您匯出交談檔案時，有註解的檢視轉換成 PDF 檔案，並下載至 [匯出] 資料夾。 PDF 版本的同一個交談檔案指向一個交談檔案中的郵件。  
  
   - **個別聊天訊息：** 當您匯出個別郵件時，交談中每個唯一的郵件會匯出為獨立的項目。 檔案會儲存如信箱所示的相同格式匯出。 針對特定的交談，您會收到多個.msg 檔案。 

     >[!NOTE]
     > 如果您套用至交談檔案的註釋，這些註釋不會傳輸至個別郵件。 

c. 其他選項

   - **產生所有匯出內容的文字檔：** 會產生匯出檢閱組中每個交談的文字檔案。 

   - **取代匯出內容與 redacted Pdf:** 如果檢閱程序期間產生 redacted 的交談檔案，然後這些檔案可在匯出期間。 您可以決定是否要匯出只有的原生檔案 （未選取此選項） 或以的原生檔案 （藉由選取此選項），這會匯出為 PDF 檔案 redacted 版本取代的原生檔案。
