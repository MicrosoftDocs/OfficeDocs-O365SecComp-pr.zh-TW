# <a name="create-a-legal-hold-notice"></a>建立的合法持有通知

組織可以使用進階電子文件 custodian 通訊，來管理他們的工作流程與 custodians 通訊。 透過 [通訊] 工具，法務小組可以有系統地傳送、 收集和追蹤合法持有通知。 彈性建立程序也能讓小組合作，來自訂保留通知工作流程與傳送到 custodians 通知中的內容。 

![通訊] 頁面](../media/CommunicationPage.PNG)

本文概述保留通知工作流程中的步驟。

## <a name="step-1-specify-communication-details"></a>步驟 1： 指定通訊詳細資料

第一個步驟是指定合法持有傳送通知適當的詳細資訊或其他 custodian 通訊。 

![名稱 [通訊] 頁面](../media/NameCommunication.PNG)

1. 在安全性 & 合規性中心，移至**eDiscovery > 進階電子文件**來顯示貴組織中的案例清單。
   
2. 按一下 [**通訊**] 索引標籤，然後按一下 [**新的通訊**。
   
3. 在 [**名稱通訊**] 頁面上，指定下列 （必要） 的通訊詳細資料。

    - **名稱**： 這是進行通訊的名稱。
    
    - **發行主管**：] 下拉式清單會顯示一份案例的成員。 傳送至 custodians 每個通知會傳送代表指定之單位法務人員。

4. 按 [下一步]****。

## <a name="step-2-define-the-portal-content"></a>步驟 2： 定義入口網站內容

接下來，您可以建立並新增保留通知的內容。 在 [**建立的通訊**] 精靈的 [**定義入口網站內容**] 頁面上，指定保留通知的內容。 此內容會自動附加至發行、 重新發出、 提醒和呈報流程通知。 此外，此內容會出現在 custodian 規範入口網站。 

![入口網站內容頁面](../media/PortalContent.PNG)

若要建立入口網站的內容：

1. 輸入 （或剪下並貼上從另一個文件） 您保留請注意，在入口網站內容的 textbox。 

2. 插入您的通知，以自訂通知及共用 Custodian 規範入口網站中的合併列印的變數。

3. 按 [下一步]****。

  >[!Tip]
  >若要深入了解更多有關如何可以自訂的內容和格式的入口網站的內容，請參閱[使用 Communications 編輯器](using-communications-editor.md)。

## <a name="step-3-set-the-required-notifications"></a>步驟 3： 設定必要的通知

您已定義保留通知的內容之後，您可以設定繞傳送和管理通知程序的工作流程。 通知會傳送通知電子郵件並與 custodians 待處理。 新增至通訊每個 custodian 會收到相同的通知。 

若要設定並傳送保留通知，您必須包含發行，重新發佈，並釋放通知。

### <a name="issuance-notification"></a>發行通知 

建立通訊之後，指定會核發法務人員所起始**發行通知**。 發行通知是傳送至 custodian 以通知他們保留義務的第一個通訊。 

若要建立發行通知：

1. 在 [**發行**] 磚中，按一下 [**編輯**]。
   
2. 如有必要，加入其他案例的成員或員工的 **[副本]** 及 **[密件副本]** 欄位。 若要將多位使用者新增至這些欄位，請使用分號分隔電子郵件地址。
   
3. 指定**主體**的 （必要） 的注意事項。
   
4. 指定的內容或您想要提供給 custodian （必要） 的其他指示。 請注意在步驟 2 中所定義的入口網站內容新增至發行通知的結尾。 
   
5. 按一下 [儲存]****。 

### <a name="re-issuance-notification"></a>重新發行通知 

大小寫的進度，custodians 可能需要比先前指示保留其他或更少的資料。 更新保留通知的內容之後，重新發行通知會提醒 custodians 其保留義務變更資訊。

若要建立重新發行通知： 

1. 在 [**重新提交**] 磚，按一下 [**編輯**]。
   
2. 如有必要，加入其他案例的成員或員工的 **[副本]** 及 **[密件副本]** 欄位。 若要將多位使用者新增至這些欄位，請使用分號分隔電子郵件地址。
   
3. 指定**主體**的 （必要） 的注意事項。
   
4. 指定的內容或您想要提供給 custodian （必要） 的其他指示。 請注意在步驟 2 中所定義的入口網站內容新增至重新發行通知的結尾。
   
5. 按一下 [儲存]****。

>[!Note]
>如果修改保留通知，重新發行通知會自動傳送給所有 custodians 指派給通知。 傳送通知之後，將要求 custodians 重新認可其保留通知。 如果您已設定的任何提醒或呈報工作流程，這些也會一併重新啟動。 

### <a name="release-notification"></a>釋放通知

專家解析後，或如果 custodian 不再主體保存內容，您可以釋放從案例 custodian。 如果 custodian 先前所發行的保留通知，釋放通知可以用於警示 custodians，他們已經發行從其義務。

若要建立釋放通知： 

1. 在 [**版本**] 磚中，按一下 [**編輯**]。
   
2. 如有必要，加入其他案例的成員或員工的 **[副本]** 及 **[密件副本]** 欄位。 若要將多位使用者新增至這些欄位，請使用分號分隔電子郵件地址。
   
3. 指定**主體**的 （必要） 的注意事項。
   
4. 指定的內容或您想要提供給 custodian （必要） 的其他指示。
   
5. 按一下 [**儲存**]，然後移至下一個步驟。 

## <a name="optional-step-4-set-the-optional-notifications"></a>（選用）步驟 4： 設定選用的通知

或者，您可以簡化遵循與回應 custodians 藉由建立及排程自動化的提醒及呈報流程通知的工作流程。

![提醒/升級] 頁面](../media/ReminderEscalations.PNG)

### <a name="reminders"></a>提醒

您已傳送的保留通知之後，您可以待處理事項與回應 custodians 所定義的提醒工作流程。 

若要排程提醒：

1. 在 [**提醒**] 磚中，按一下 [**編輯**]。
   
2. 藉由開啟 （必要）**狀態**切換啟用**提醒**工作流程。
   
3. 指定**提醒時間間隔 （天）** （必要）。 這是傳送第一個及待處理的提醒通知前所等待的天數。 例如，如果您將提醒間隔設定為 7 天，然後第一個提醒將會傳送 7 天後保留通知最初發行。 所有後續提醒會也會傳送每隔 7 天。
   
4. 指定**的提醒數目**（必要）。 此欄位會指定要傳送給未回應 custodians 多少提醒。 例如，如果您設定的提醒數目設為 3，然後 custodian 會收到最大值為 3 的提醒。 Custodian 認可保留通知之後，該使用者將不再傳送提醒。
   
5. 指定**主體**的 （必要） 的注意事項。 
   
6. 指定的內容或您想要提供給 custodian （必要） 的其他指示。 請注意在步驟 2 中所定義的入口網站內容會新增至提醒通知的結尾。
   
7. 按一下 [**儲存**]，然後移下一個步驟。

### <a name="escalations"></a>擴大 

在某些情況下，您可能需要與回應 custodians 的待處理的其他方法。 如果 custodian 不會認可收到指定的提醒數目之後保留通知，法律小組可以指定自動將呈報流程通知傳送給 custodian 和其管理員的工作流程。

若要排程擴大：

1. 在 [**升級**] 磚，按一下 [**編輯**]。
   
2. 藉由開啟**狀態**切換啟用**呈報**工作流程。
   
3. 指定**升級時間間隔 （天）** （必要）。 
   
4. 指定**擴大數目**（必要）。 此欄位會指定要傳送給未回應 custodians 多少擴大。 例如，如果您將擴大數目設定為 3，然後呈報流程通知會送給 custodian 和其主管最大值為 3 次。 Custodian 認可保留通知之後，將不再傳送擴大。 
   
5. 指定**主體**的 （必要） 的注意事項。 
   
6. 指定的內容或您想要提供給 custodian （必要） 的其他指示。 請注意在步驟 2 中所定義的入口網站內容會新增至呈報流程通知的結尾。
   
7. 按一下 [**儲存**]，然後移下一個步驟。
   
## <a name="step-5-assign-custodians"></a>步驟 5： 指派 custodians 

您已完成通知的內容之後，選取您想要將通知傳送 custodians。 

![選取 Custodians 頁面](../media/SelectCustodians.PNG)

若要新增 custodians:

1. 指派給通訊 custodians 按一下其名稱旁邊的核取方塊。

    建立通訊之後，通知工作流程會自動套用到選取 custodians。
   
2. 按 [**下一步**檢閱通訊設定與詳細資料。
 
>[!NOTE]
>您只能加入 custodians 已新增至案例，尚未傳送案例內的其他通知。

## <a name="step-6-review-settings"></a>步驟 6： 檢閱設定

檢閱設定，然後按一下 [**傳送**至完成通訊之後，系統會自動啟動通訊工作流程所傳送的發行明。
