# <a name="create-a-legal-hold-notice"></a>建立的合法持有通知

組織可以使用進階的 eDiscovery （預覽） okay 通訊，來管理周圍與 custodians 通訊時發生其工作流程。透過通訊工具法律小組可以詢問傳送、 收集和追蹤合法持有通知。具彈性的建立程序也可讓小組來自訂保留通知工作流程與傳送到 custodians 通知中的內容。 

本文概述保留通知工作流程中的步驟。

## <a name="step-1-specify-communication-details"></a>步驟 1： 指定通訊的詳細資訊

第一個步驟是指定適當的詳細資訊的合法持有通知或其他 okay 通訊。 

1. 在安全性 & 規範中心中，移至 [ **eDiscovery > 進階的 eDiscovery （預覽）** 若要在組織中顯示的情況下的清單。
   
2. [**通訊**] 索引標籤和 [**新的通訊**。
   
3. **名稱的通訊**] 索引標籤上指定下列 （必要） 通訊的詳細資訊。

    - **名稱**： 這是進行通訊的名稱。
    
    - **發行主管**： 下拉式清單會顯示 case 成員的清單。每個通知傳送給 custodians 將傳送代表指定之發行主管人員。

4. 按 [下一步]。

## <a name="step-2-define-the-portal-content"></a>步驟 2： 定義入口網站的內容

接下來，您可以建立及新增保留通知的內容。在 [**建立的通訊**] 精靈中的 [**定義入口網站內容**] 頁面上指定的保留通知的內容。此內容會自動附加至發行、 重新問題、 提醒、 及呈通知。此外，此內容會出現在 okay 規範入口網站。 

若要建立入口網站的內容：

1. 類型 （或剪下並從另一個文件 paster） 您保留注意到 textbox 之入口網站的內容中。 

2. 合併變數插入您的通知，以自訂通知及共用 Okay 規範入口網站。

3. 按 [下一步]。

  >[!Tip]
  >若要了解更多有關可自訂的內容與格式的入口網站的內容，請參閱[使用通訊編輯器](using-communications-editor.md)。

## <a name="step-3-set-the-required-notifications"></a>步驟 3： 設定必要的通知

您已定義的保留明內容之後，您可以設定繞傳送和管理通知程序的工作流程。通知會傳送通知的電子郵件及待處理與 custodians。新增至通訊每 okay 會收到相同的通知。 

若要設定並傳送保留通知，您必須包含發行、 Re-發行和釋出的通知。

### <a name="issuance-notification"></a>發行通知 

建立通訊之後，請指定會核發人員所起始**發行通知**。發行通知是第一個通訊會傳送到 okay 用以告知他們有關其保留責任。 

若要建立發行通知：

1. 在 [**發行**] 磚中，按一下 [**編輯**]。
   
2. 如有必要，新增其他案例的成員或人員到 **[副本]** 及 **[密件副本]** 欄位。將多位使用者新增至這些欄位，請使用分號分隔電子郵件地址。
   
3. 指定**主體**的 （必要） 的通知。
   
4. 指定的目錄或其他您想要提供給 （必要） okay 的指示。請注意您在步驟 2 中所定義的入口網站內容已新增至發行通知的結尾。 
   
5. 按一下 [儲存]**** 

### <a name="re-issuance-notification"></a>回覆發行通知 

Case 隨著 custodians 可能需要比先前指示保留其他或更少的資料。您要更新的保留通知的內容後，re 發行通知的提醒 custodians 其保留義務變更資訊。

若要建立此回覆發行通知： 

1. 在 [**重新提交**] 磚中按一下 [**編輯**]。
   
2. 如有必要，新增其他案例的成員或人員到 **[副本]** 及 **[密件副本]** 欄位。將多位使用者新增至這些欄位，請使用分號分隔電子郵件地址。
   
3. 指定**主體**的 （必要） 的通知。
   
4. 指定的目錄或其他您想要提供給 （必要） okay 的指示。請注意您在步驟 2 中所定義的入口網站內容已新增至 re 發行通知的結尾。
   
5. 按一下 [儲存]****。

>[!Note]
>如果已修改的保留通知，re 發行通知將會自動傳送給所有 custodians 指派給通知。就會傳送通知後，custodians 系統會要求重新確認其保留另行通知。如果您已經設定任何提醒或呈報工作流程，這些會一併重新啟動。 

### <a name="release-notification"></a>釋放通知

專家解析之後或 okay 不再限於保留內容，您可以釋放從案例 okay。如果 okay 先前已發出的保留通知，釋放通知可用來警示 custodians 他們有已從其義務發行。

若要建立釋放通知： 

1. 在 [**版本**] 磚中按一下 [**編輯**]。
   
2. 如有必要，新增其他案例的成員或人員到 **[副本]** 及 **[密件副本]** 欄位。將多位使用者新增至這些欄位，請使用分號分隔電子郵件地址。
   
3. 指定**主體**的 （必要） 的通知。
   
4. 指定的目錄或其他您想要提供給 （必要） okay 的指示。
   
5. 按一下 [**儲存**並移至下一個步驟。 

## <a name="optional-step-4-set-the-optional-notifications"></a>（選用）步驟 4： 設定選用的通知

（選用） 您可以簡化遵循與回應 custodians 藉由建立及排程自動的提醒及呈通知的工作流程。

### <a name="reminders"></a>提醒

已傳送保留通知之後，您可以待處理與回應 custodians 所定義的提醒工作流程。 

若要排程提醒：

1. 在 [**提醒**] 磚中，按一下 [**編輯**]。
   
2. 啟用 （必要）**狀態**切換開啟**提醒**工作流程。
   
3. 指定**提醒的時間間隔 （以天數）** （必要）。這是傳送第一個及後續的提醒通知前所等候的天數。例如，如果您將提醒時間間隔設定為 7 天，然後第一個提醒會傳送 7 天後保留通知最初發行。所有後續提醒就也傳送每 7 天。
   
4. 指定**的提醒數目**（必要）。此欄位會指定將傳送至未回應 custodians 多少提醒。例如，如果您將提醒數目設定為 3，然後 okay 才會收到最大值為 3 的提醒。Okay 認可保留通知後，該使用者將無法再傳送提醒。
   
5. 指定**主體**的 （必要） 的通知。 
   
6. 指定的目錄或其他您想要提供給 （必要） okay 的指示。請注意您在步驟 2 中所定義的入口網站內容已新增至提醒通知的結尾。
   
7. 按一下 [**儲存**並移下一個步驟。

### <a name="escalations"></a>呈報 

在某些情況下，您可能需要使用回應 custodians 的待處理的額外方式。如果 okay 不會確認之後收到提醒的指定的數目的保留通知、 法律小組可以指定為自動將升級通知傳送至 okay 和其管理員的工作流程。

若要排程呈報：

1. **呈報**並排顯示中按一下 [**編輯**]。
   
2. 啟用**呈報**流程開啟**狀態**切換。
   
3. 指定**呈報間隔 （以天數）** （必要）。 
   
4. 指定**呈報數目**（必要）。此欄位會指定將傳送至未回應 custodians 多少呈報。例如，如果您將呈報數目設定為 3，然後升級通知嗎傳送 okay 與及其主管最大值為 3 次。Okay 認可保留通知之後，將不會再傳送呈報。 
   
5. 指定**主體**的 （必要） 的通知。 
   
6. 指定的目錄或其他您想要提供給 （必要） okay 的指示。請注意您在步驟 2 中所定義的入口網站內容已新增的升級通知的結尾。
   
7. 按一下 [**儲存**並移下一個步驟。
   
## <a name="step-5-assign-custodians"></a>步驟 5： 指派 custodians 

您已完成的通知內容之後，選取 [傳送至通知 custodians。 

若要新增 custodians：

1. 指派至通訊 custodians 按一下其名稱旁邊的核取方塊。

    建立通訊之後，通知工作流程會自動將套用至所選 custodians。
   
2. 按 [**下一步**可供檢閱的通訊設定和詳細資料。
 
>[!NOTE]
>您只可以新增 custodians 有已新增案例及尚未尚未傳送案例中的其他通知。

## <a name="step-6-review-settings"></a>步驟 6： 檢閱設定

檢閱設定並按一下 [完成將通訊**傳送**之後，系統會自動啟動通訊工作流程所傳送的發行通知。
