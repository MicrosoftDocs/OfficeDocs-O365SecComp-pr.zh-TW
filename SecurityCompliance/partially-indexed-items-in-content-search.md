# <a name="partially-indexed-items-in-content-search-in-office-365"></a>位於 Office 365 中內容搜尋的已局部編製索引項目

您執行從 Office 365 安全性內容搜尋&amp;規範中心自動包含部分已編製索引的項目中預估的搜尋結果時執行搜尋。部分已編製索引的項目是 Exchange 信箱項目及 SharePoint 和 OneDrive 上的商務網站因任何原因未完全編入搜尋索引的文件。Exchange 中的部分已編製索引的項目通常包含檔案-無法編製索引的檔案類型 — 會附加至電子郵件訊息。以下是其他項目無法編製索引的搜尋及原因當您執行搜尋時所傳回的部分索引項目： 
  
- 此檔案類型是無法辨識或不支援編製索引。
    
-  郵件已如圖像檔 ； 有效的處理常式中，而附加的檔案這是最常見原因的部分已編製索引的電子郵件項目。 
    
- 檔案類型支援索引編製，但在編製特定檔案的索引時發生錯誤。
    
- 附加到電子郵件的檔案太多。
    
- 附加到電子郵件的檔案太大。
    
- 檔案已使用非 Microsoft 技術加密。
    
- 檔案受密碼保護。
    
> [!NOTE]
> 大部分的 Office 365 組織具有的內容低於 1%大量然後小於 12%以部分編製索引的大小。磁碟區及大小之間的差異的原因是較大的檔案具有較高的機率的含有不能完全索引的內容。 
  
法律調查，針對您的組織可能需要檢閱部分已編製索引的項目。您也可以指定是否要包含部分已編製索引的項目時將搜尋結果匯出到本機電腦或結果做好使用 Office 365 進階 eDiscovery 分析。如需詳細資訊，請參閱[Investigating 部分編製索引的 Office 365 在 eDiscovery 中的項目](investigating-partially-indexed-items-in-ediscovery.md)。
  
## <a name="file-types-not-indexed-for-search"></a>未編製索引以用於搜尋的檔案類型

特定類型的檔案，例如點陣圖或 MP3 檔案不包含內容編製索引。因此，搜尋 Exchange 中的索引伺服器與 SharePoint 未執行全文檢索索引在這些類型的檔案。這些類型的檔案被視為不支援的檔案類型。也有其全文檢索索引已停用，預設或由系統管理員的檔案類型。不支援但已停用檔案類型會標示為在內容搜尋中編製索引項目。如前所述、 部分已編製索引的項目可以包含在搜尋結果集中執行搜尋時，將搜尋結果匯出至本機電腦或準備進階 eDiscovery 搜尋結果。 
  
如需所支援和已停用的檔案格式清單，請參閱下列主題︰
  
- **Exchange** - [依 Exchange 搜尋編製索引的檔案格式](https://go.microsoft.com/fwlink/p/?LinkID=386618)
    
- **Exchange** - [Get-searchdocumentformat](https://go.microsoft.com/fwlink/p/?LinkID=724037)
    
- **SharePoint** - [預設編目檔案副檔名及剖析 SharePoint 中的檔案類型](https://go.microsoft.com/fwlink/p/?LinkID=404033)
    

  
## <a name="messages-and-documents-with-partially-indexed-file-types-can-be-returned-in-search-results"></a>郵件和文件部分編製索引的檔案可以搜尋結果中傳回的類型

為部分已編製索引的項目自動傳回具有部分已編製索引的檔案附件不是每封電子郵件或每個部分已編製索引的 SharePoint 文件。這是因為其他訊息或電子郵件訊息中**Subject**之類的文件屬性及文件的**標題**或**作者**的內容編製索引並可供搜尋。例如，「 財務"關鍵字搜尋會傳回具有部分已編製索引的檔案附件的項目如果該關鍵字出現在電子郵件的主旨或在檔案名稱或文件標題。不過，如果關鍵字只會出現在本文的檔案、 訊息或文件會傳回為部分已編製索引的項目。 
  
同樣地，部分已編製索引的檔案附件和部分已編製索引的檔案類型的文件的郵件並包含搜尋結果時其他訊息或文件屬性，亦即已編製索引和搜尋、 符合搜尋準則。編製索引的搜尋編製郵件內容包括郵件內文中的傳送和接收日期、 寄件者和收件者的附件，而且文字的檔案名稱。搜尋編製索引的文件內容] 包括建立和修改日期。那麼即使為郵件附件可能會局部編製索引的項目，郵件將會包含一般的搜尋結果中，如果其他訊息或文件屬性的值符合搜尋準則。
  
如需您可以使用安全性的搜尋功能來搜尋的電子郵件和文件屬性的清單&amp;規範中心，請參閱[關鍵字查詢和搜尋條件的內容搜尋](keyword-queries-and-search-conditions.md)。
  

  
## <a name="partially-indexed-items-included-in-the-search-results"></a>在搜尋結果中包含的部分已編製索引項目
<a name="unindexeditemsresults"> </a>

您的組織可能就需要識別並執行其他分析判斷及其、 什麼它們包含，以及是否它們與特定調查相關的部分已編製索引項目。如先前所述的部分已編製索引的項目中要搜尋的內容位置隨附自動預估的搜尋結果。您有這些部分已編製索引的項目時所要包含匯出搜尋結果或準備進階 eDiscovery 搜尋結果] 選項。部分已編製索引的項目時所要包含您要匯出搜尋結果或其準備進階 ediscovery （英文）、 選取其中一個選項，包括具有無法辨識的格式、 加密，或未編製索引的其他原因的項目。
  
記住下列關於部分已編製索引的項目：
  
- 當您執行內容的搜尋時，總計的數目與大小部分已編製索引的項目 （搜尋查詢所傳回） 會顯示在 [詳細資料] 窗格中的搜尋統計資料標示為 「 編製索引項目 」。
    
- 如果您選取 [下載做為 Exchange 項目] 選項時您匯出搜尋結果並包含部分已編製索引的項目，請將 Exchange 的部分索引項目都要匯出至不同 PST 檔案中其所，或以個別郵件的信箱郵件。SharePoint 的部分索引項目都要匯出至名為**Uncrawlable**的資料夾。
    
- 如果您要匯出結果的搜尋特定的內容位置或組織中的所有內容位置的搜尋，只包含符合搜尋準則的項目的內容位置編製索引項目將會匯出。換句話說，如果沒有搜尋結果中的信箱或站台，然後該信箱或網站的任何編製索引項目將不會匯出。此原因是匯出部分已編製索引的項目從許多組織中的位置可能會增加匯出錯誤的機率並增加匯出及下載的搜尋結果所花費的時間。
    
    若要從搜尋的所有內容位置匯出部分已編製索引的項目、 設定搜尋以傳回所有項目 （搜尋查詢中移除任何關鍵字），然後將匯出只部分索引項目 （藉由按一下**只匯出搜尋結果時無法辨識的格式，項目已加密或未編製索引的其他原因****輸出] 選項**下)。
    
- 如果您選擇將搜尋結果中包含所有的信箱項目或搜尋查詢不指定任何關鍵字或只指定日期範圍，部分已編製索引的項目可能無法複製到包含部分已編製索引的項目 PST 檔案。這是因為包括任何部分已編製索引的項目、 的所有項目將會自動包含規則的搜尋結果中。
    
- 部分已編製索引的項目都無法提供要預覽。您必須將檔案匯出搜尋結果，以檢視搜尋傳回的部分已編製索引項目。

## <a name="partially-indexed-items-excluded-from-the-search-results"></a>從搜尋結果中排除的部分已編製索引項目

如果部分編製索引項目，但它不會符合搜尋查詢的準則，它不會納入為搜尋結果中的部分已編製索引項目。換句話說，從搜尋結果排除項目。例如，假設您執行搜尋並不包含任何關鍵字或屬性因為您想要包含的所有內容。但包含查詢的日期範圍條件。如果部分已編製索引的項目都屬於該日期範圍，它不會納入為部分已編製索引的項目。日期範圍是有效的方式來排除部分已編製索引的項目從搜尋結果。
  
同樣地，如果您選擇包含部分已編製索引的項目匯出搜尋結果時，將不會匯出已從搜尋結果中排除的部分已編製索引項目。
  
此規則的一個例外是當您建立查詢式保留與 eDiscovery 案例相關聯。如果您建立查詢式保留，部分已編製索引的所有項目被處於保留狀態。這包括部分已編製索引的項目不符合搜尋查詢準則和部分已編製索引的項目可能屬於以外的日期範圍條件。如保留建立查詢為基礎的詳細資訊，請參閱步驟 4 中[的 [Office 365 安全性及規範中心的 eDiscovery 案例](ediscovery-cases.md#step-4-place-content-locations-on-hold)。

限制 SharePoint 文件編製索引的清單，請參閱[SharePoint Online 的搜尋限制](https://support.office.com/article/7c06e9ed-98b6-4304-a900-14773a8fa32f)。

## <a name="more-information-about-partially-indexed-items"></a>部分已編製索引的項目相關的詳細資訊
<a name="moreinfo"> </a>

- 先前所述，因為郵件和文件的屬性和其中繼資料編製索引，關鍵字搜尋可能傳回的結果如果該關鍵字會出現在已編製索引的中繼資料。但是，該相同的關鍵字搜尋可能會傳回相同的項目如果關鍵字才會顯示含有不受支援的檔案類型的項目內容中。在此例中項目會傳回為部分已編製索引的項目。
    
- 如果因為它符合搜尋查詢準則 （且） 排除） 部分已編製索引的項目包含在搜尋結果中然後它將包含為預估的搜尋統計資料的部分已編製索引項目。此外，它將不會是包含在部分已編製索引的項目匯出搜尋結果時。
    
- 雖然檔案類型支援索引編製及已編製索引，可以有索引或搜尋會導致檔案以部分已編製索引的項目傳回的錯誤。例如，搜尋非常大的 Excel 檔案可能會局部成功 （因為第 4 MB 編製索引），但然後會因為超出檔案大小限制而失敗。在此例中有可能與搜尋結果] 和 [存部分已編製索引的項目傳回的同一個檔案。
    
- 使用 Microsoft 技術加密的附加的檔案編製索引和可搜尋。使用非 Microsoft 技術加密的檔案部分編製索引。
    
- 使用 S/MIME 加密的電子郵件訊息部分編製索引。這包括使用或不含檔案附件的加密的郵件。
    
- 使用資訊版權管理 (IRM) 保護的郵件會編製索引，因此如果符合搜尋查詢，便會將它們納入搜尋結果。
