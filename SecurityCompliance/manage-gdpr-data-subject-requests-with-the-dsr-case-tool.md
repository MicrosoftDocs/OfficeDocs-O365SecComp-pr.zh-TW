# <a name="manage-gdpr-data-subject-requests-with-the-dsr-case-tool-in-the-office-365-security-amp-compliance-center"></a>管理 Office 365 安全性 DSR case 工具 GDPR 資料主體要求&amp;規範中心

歐盟一般資料保護法規 (GDPR) 是關於保護及啟用個人隱私權版權內歐盟 （歐盟）。歐盟 GDPR 提供個人 （又稱為資料主旨） 來存取、 擷取、 校正、 清除，並限制處理其個人資料的右邊。下 GDPR 個人資料表示已識別或識別自然人員與相關的任何資訊。正式要求所組織的人員在其個人資料採取動作稱為資料主體要求或 DSR。如需 Office 365 中的資料回應 DSRs 的詳細資訊，請參閱[Office 365 資料主體要求的指南](https://go.microsoft.com/fwlink/?linkid=871169 )。
  
若要管理組織中的人員送出 DSR 回應調查，您可以使用 DSR case 工具 Office 365 安全性&amp;規範中心以尋找中儲存的內容：
  
- 在組織中所有使用者信箱。這包括 Skype 商務交談和一對一的 Microsoft 小組中聊天
    
- 與 Office 365 群組及 Microsoft 小組中的所有小組信箱相關聯的所有信箱
    
- 貴組織中所有的 Sharepoint 網站和商務用 OneDrive 帳戶
    
- 所有小組網站與您組織中的 Office 365 小組網站
    
- Exchange Online 中的所有公用資料夾
    
您可以使用 DSR case 工具：
  
- 針對每一個 DSR 調查建立個別案例。
    
- 管制哪些人可以透過新增成員的大小寫 ； 人員 DSR 大小寫的存取只有成員可以存取案例和只能在 [安全性] **DSR 的情況下**] 頁面上看到其清單中的情況下的情況下&amp;規範中心。此外，您可以將不同的權限指派給的同一個案例的不同成員。例如，您可以讓某些成員僅限檢視案例和搜尋結果及允許其他成員建立搜尋及匯出搜尋結果。 
    
- 內建搜尋可用於建立或上傳的特定資料主體的所有內容。
    
- （選用） 修改內建的搜尋查詢並重新執行搜尋來縮小搜尋結果。
    
- 新增其他 DSR 案例相關聯的內容搜尋。這包括建立從 「 我的分析及 Office 漫遊服務傳回部分已編製索引的項目及系統產生的記錄檔的搜尋。
    
- 匯出資料以回應 DSR 存取或匯出要求。
    
- 完成; DSR 調查程序時刪除案例這會移除所有搜尋並匯出與案例相關聯的工作。

## <a name="step-1-assign-ediscovery-permissions-to-potential-case-members"></a>步驟 1：將 eDiscovery 權限指派給潛在的案例成員

根據預設，Office 365 全域管理員可以存取安全性 DSR case 工具&amp;規範中心。根據設計，例如資料隱私權主管、 人力資源管理員中，其他使用者或參與 DSR 調查的其他人沒有存取權 DSR case 工具和要指派給適當的權限來存取此工具會有。為達成此目的最簡單的方法是移至 [安全性]**權限**] 頁面&amp;規範中心並將使用者新增至 eDiscovery 管理員角色群組。請注意您也可以讓您可以將其新增為您在步驟 2 中建立 DSR 案例成員指派這些權限。 
  
如需逐步說明，請參閱[指派 Office 365 安全性 eDiscovery 權限&amp;規範中心](assign-ediscovery-permissions.md)。
  
> [!NOTE]
> 根據預設，Office 365 全域管理員 (或其他組織管理角色群組成員的安全性&amp;規範中心沒有匯出內容的搜尋結果 （請參閱本文中的步驟 4） 的必要權限。若要解決這個問題，系統管理員可以新增自己做 eDiscovery 管理員角色群組的成員。

## <a name="step-3-run-the-search-query"></a>步驟 3： 執行搜尋查詢

建立 DSR 案例及新增成員之後下, 一步是執行內建的 [搜尋與案例相關聯。此預設搜尋查詢會執行下列動作：
  
- 在您的組織的所有電子郵件項目已傳送或接收資料之主旨中搜尋所有信箱。這被藉由使用搜尋資料主體電子郵件中的所有人欄位中的*參與者*電子郵件屬性。這個屬性會傳回在其資料主體是**從**、 **To**、 **[副本**] 和 **[密件副本]** 欄位中的項目。Exchange Online 中的公用資料夾也會搜尋訊息傳送或接收資料主體。 
    
- 搜尋文件和建立或上傳的資料之主旨的項目在組織中的所有網站。這被藉由使用下列網站內容：
    
  - *作者*屬性會傳回的項目其中資料主體會列在 Office 文件中的 [作者] 欄位。此值持續發生，即使複製並由其他人上傳文件。 
    
  - *CreatedBy*屬性會傳回已建立或上傳的資料之主旨的項目。 
    
以下是內建取得自動建立時建立 DSR 案例的搜尋關鍵字查詢外觀。
  
```
participants:"<email address>" OR author:"<display name>" OR createdby:"<display name>"
```

例如，若為 Leonte 資料主體名稱，關鍵字查詢起來如下：
  
```
participants:"ina@contoso.com" OR author:"Ina Leonte" OR createdby:"Ina Leonte"
```

## <a name="step-4-export-the-data"></a>步驟 4： 將資料匯出

執行內建搜尋之後，您可以將匯出的搜尋結果。或者，匯出資料之前，您可能會想要修改的查詢來減少的搜尋結果數目。如需縮小搜尋結果的詳細資訊，請參閱步驟 5。
  
當您將搜尋結果匯出信箱項目可以下載 PST 檔案中或以個別的郵件。當您從 SharePoint 和 OneDrive 帳戶匯出內容時，都要匯出的原生 Office 文件和其他文件複本。結果包含的檔案，匯出每個項目的相關資訊也會包含在搜尋結果。如需詳細資訊匯出，請參閱[匯出內容的搜尋結果的 Office 365 安全性&amp;規範中心](export-search-results.md)。
  
> [!NOTE]
> 根據預設，Office 365 全域管理員 (或其他組織管理角色群組成員的安全性&amp;規範中心) 不需要將內容的搜尋結果匯出的必要權限。若要解決這個問題，系統管理員可以新增自己做 eDiscovery 管理員角色群組的成員。 
  
您用來匯出資料的電腦必須符合下列系統需求：
  
- Windows 7 和更新版本的 32 或 64 位元版本



    
- Microsoft.NET Framework 4.7
    
- 支援的瀏覽器：
    
  - Microsoft Edge
    
    或
    
  - Microsoft Internet Explorer 10 或更新版本
    
    > [!NOTE]
    > Microsoft 不會製造協力廠商擴充功能或 ClickOnce 應用程式的附加元件。匯出資料與協力廠商擴充功能或附加元件使用不受支援的瀏覽器不支援。

## <a name="optional-step-5-revise-the-built-in-search-query"></a>（選用）步驟 5： 修改內建的搜尋查詢

執行內建搜尋之後，您可以修改它縮小傳回較少的搜尋結果的範圍。您可以新增至查詢的條件。條件是以邏輯方式連線到關鍵字查詢**AND**運算子。這表示要傳回的搜尋結果中的項目必須滿足的關鍵字查詢和新增任何條件。這是條件，縮減結果協助的方式。如果您將兩個或多個唯一的條件新增至搜尋查詢 （指定不同的屬性的條件） 時，這些條件就必定連線所**AND**運算子。這表示會傳回只滿足 （除了關鍵字查詢） 的所有條件的項目。如果您將多個值 （以逗號或分號分隔） 新增至單一條件時，這些值是透過**OR**運算子連接。這表示如果它們包含任何指定之條件中屬性的值會傳回的項目。 
  
以下是一些您可以新增至內建的搜尋查詢 DSR 案例的情況的範例。在搜尋查詢中使用的實際屬性的名稱會顯示括弧括住。
  
- **檔案類型 ( `filetype`)** -指定文件或檔案的副檔名。使用此條件可搜尋文件與特定的 Office 應用程式，例如 Word、 Excel 及 OneNote 所建立的檔案。 
    
- **訊息類型 ( `kind`)** -會指定要搜尋的電子郵件項目的類型。例如，您可以使用下列的語法`kind:email OR kind:im`以便只傳回電子郵件和 Skype 商務交談或一對一聊天室中的 Microsoft 小組。 
    
- **規範標記 (`compliancetag`)** -指定指派給電子郵件訊息或文件的標籤。此條件會傳回特定的標籤與分類的項目。標籤可用來分類電子郵件和文件資料管理和強制執行保留標籤所定義的分類為基礎的規則。這會是 DSR 調查的實用條件是因為您的組織可能會使用標籤來分類資料隱私權相關的內容或包含個人資料或機密資訊。此條件的值，使用含萬用字元的標籤名稱的第一個部分或完整的標籤名稱。如需詳細資訊，請參閱 ＜ [Overview of Office 365 中的標籤](labels.md)。
    
清單及說明可用於 DSR case 工具中的所有條件，請參閱 「 關鍵字查詢和搜尋條件的內容搜尋 」 文章中的[搜尋條件](keyword-queries-and-search-conditions.md#search-conditions)。

### <a name="searching-exchange-public-folders"></a>搜尋 Exchange 公用資料夾

DSR 案例中內建的搜尋將只傳回電子郵件資料主體傳送給擁有郵件功能的公用資料夾或其他人傳送給公用資料夾和也複製資料之主旨的郵件。它不會傳回資料主體可能會有張貼的訊息至公用資料夾。若要搜尋資料主體張貼至公用資料夾的項目，您可以建立不同建立不同的內容搜尋會搜尋張貼至公用資料夾資料之主旨的任何項目。
  
以下是整體程序，以搜尋資料主體可能會有張貼至公用資料夾的項目。 
  
1. 開啟 DSR 案例，並在 [**搜尋**] 頁面上建立新的搜尋。 
    
2. 使用下列準則設定搜尋查詢及內容的位置來搜尋：
    
  - 在 [**關鍵字**] 方塊中，使用下列的搜尋查詢： 
    
    ```
    itemclass:ipm.post AND "<email address of the data subject>"
    ```

  - 搜尋所有的 Exchange 公用資料夾
    
  - 執行搜尋，並完成之後，您可以匯出並下載搜尋結果 （如[步驟 4](manage-gdpr-data-subject-requests-with-the-dsr-case-tool.md#step4)中所述）。使用下列設定值匯出部分已編製索引的項目。
