# <a name="set-up-an-archive-and-deletion-policy-for-mailboxes-in-your-office-365-organization"></a>設定 Office 365 組織中信箱的封存和刪除原則

 在 Office 365 系統管理員可以建立封存與刪除原則會自動將項目移至使用者的封存信箱，自動從信箱中刪除項目。 系統管理員會執行此動作所建立的保留原則指派給信箱，並將項目移至使用者的封存信箱之後一段時間，也會刪除項目從信箱後到達特定存留期限制。 實際的規則，判斷哪些項目已移動或刪除，當發生這種情況時所呼叫的保留標記。 保留標記連結至保留原則，接著是指派給使用者的信箱。 保留標記會將保留設定套用至個別郵件與使用者的信箱中的資料夾。 它會定義多久一則訊息仍會保留在信箱，以及當郵件達到指定的保留時間時不會採取什麼動作。 當郵件達到保留時間時，則可以移至使用者的封存信箱，或將它刪除。 
  
本文中的步驟，將會設定名為 [高山屋虛構組織的封存和保留原則。 設定此原則包含下列工作：
  
- 讓組織中每個使用者的封存信箱。 這讓使用者可以新增信箱儲存空間，且需要使保留原則可以將項目移至封存信箱。 它也讓我們所移動的使用者存放區保存資訊項目到其封存信箱。 
    
- 建立三個自訂的保留標記，請執行下列動作： 
    
  - 會自動移至使用者的封存信箱 3 年的項目。 將項目移至封存信箱會釋放使用者的主要信箱中的空間。
    
  - 自動刪除舊的 5 年是從 [刪除的項目] 資料夾的項目。 這也會釋放使用者的主要信箱中的空間。 使用者有機會復原這些項目，如有必要。 如需詳細資訊的[詳細資訊](#more-information)一節中的註腳，請參閱。 
    
  - 自動 （和永久） 會刪除 7 年舊從這兩個主要和封存信箱的項目。 合規性法規，因為某些組織所需的一段時間保留電子郵件。 這段時間逾期之後，組織可能會想要永久移除這些項目的使用者信箱。 
    
- 建立新的保留原則，並將新的自訂保留標籤加入至其中。 此外，您也會將內建的保留標記新增至新的保留原則。 這包括個人標記，使用者可以將指派給他們的信箱中的項目。 您也將新增將項目從使用者的主要信箱中的 [可復原的項目] 資料夾移到其封存信箱中 [可復原的項目] 資料夾的保留標記。 這可協助釋出空間使用者 [可復原的項目] 資料夾中他們的信箱處於保留狀態時。
    
您可以遵循部分或全部本篇文章以設定您的組織中信箱的封存和刪除原則] 中的步驟。 我們建議您測試幾個信箱上的此程序，再實作您的組織中所有信箱。

-  當您在 Office 365 中建立新的使用者帳戶，並對使用者指派 Exchange Online 授權時，信箱會自動建立的使用者。 建立信箱時，它已自動指派預設保留原則，名為 「 預設 MRM 原則。 在本文中，您會建立新的保留原則，並再將其指派給使用者的信箱，取代預設 MRM 原則。 信箱只能有一個一次指派給它的保留原則。

## <a name="optional-step-5-run-the-managed-folder-assistant-to-apply-the-new-settings"></a>（選用）步驟 5： 執行受管理的資料夾助理員套用新的設定

您將新的保留原則套用至步驟 4 中的信箱之後，它可能需要 7 天在 Exchange Online 新的保留設定套用至信箱。 這是因為程序稱為 「 受管理的資料夾助理員處理信箱一次每隔 7 天。 而不是等待受管理的資料夾助理員執行，您可以強制此選項，即可執行**Start-managedfolderassistant**指令程式在 Exchange Online PowerShell，即會發生。 
  
 **當您執行受管理的資料夾助理員，會發生什麼事？** 它會藉由檢查信箱中的項目，以及決定是否在受限於保留套用保留原則中的設定。 它再以適當的保留標記，戳記受限於保留的項目，然後超過保留時間的項目採取指定的保留動作。 
  
以下是連線至 Exchange Online PowerShell 中，然後執行您的組織中的每個信箱中的 [受管理的資料夾助理員的步驟。
  
1. 在您的本機電腦上開啟 Windows PowerShell，並執行下列命令。
    
    ```
    $UserCredential = Get-Credential
    ```

    在 [ **Windows PowerShell 認證要求**] 對話方塊中，輸入使用者名稱與您的 Office 365 全域系統管理員帳戶的密碼，然後按一下 [**確定]**。
    
2. 執行下列命令。
    
    ```
    $Session = New-PSSession -ConfigurationName Microsoft.Exchange -ConnectionUri https://outlook.office365.com/powershell-liveid/ -Credential $UserCredential -Authentication Basic -AllowRedirection
    ```

3. 執行下列命令。
    
    ```
    Import-PSSession $Session
    ```

4. 若要確認您已連線至 Exchange Online 組織，執行下列命令以取得貴組織中所有信箱的清單：
    
    ```
    Get-Mailbox
    ```

    > [!NOTE]
    > 如需詳細資訊或如果您有無法連線到 Exchange Online 組織，請參閱 < <b0>Connect to Exchange Online PowerShell</b0>。 
  
5. 執行下列兩個命令，以啟動受管理的資料夾助理員所有使用者信箱的組織中。
    
    ```
    $Mailboxes = Get-Mailbox -ResultSize Unlimited -Filter {RecipientTypeDetails -eq "UserMailbox"}
    ```

    ```
    $Mailboxes.Identity | Start-ManagedFolderAssistant
    ```

這就對了！ 您已設定高山屋組織封存和刪除原則。
  
## <a name="optional-step-6-make-the-new-retention-policy-the-default-for-your-organization"></a>（選用）步驟 6： 將新的保留原則貴組織的預設值

在 [步驟 4 中，您必須將新的保留原則指派給現有信箱。 但是，您可以設定 Exchange Online 讓新的保留原則指派給在未來建立的新信箱。 要這麼做，使用 Exchange Online PowerShell 來更新您的組織預設信箱計劃。 *信箱計劃*是會自動在新的信箱設定屬性的範本。  在此選用的步驟中，您可以取代目前的保留原則指派給信箱計劃 （根據預設，預設 MRM 原則），與您在步驟 3 中建立的保留原則。 更新信箱計劃之後，新的保留原則將指派給新的信箱。

1. <b0>PowerShell 連線到 Exchange Online</b0>或請參閱 < 步驟 5。

2. 執行下列命令，以顯示貴組織中的信箱計劃的相關資訊。

    ```
    Get-MailboxPlan | Format-Table DisplayName,RetentionPolicy,IsDefault
    ```
    請注意設為預設信箱計劃。

3. 執行下列命令，以將您在步驟 3 （例如，**高山屋封存和保留原則**） 中建立新保留原則指派到預設信箱計劃中。 本範例假設預設信箱計劃的名稱為**ExchangeOnlineEnterprise**。

    ```
    Set-MailboxPlan "ExchangeOnlineEnterprise" -RetentionPolicy "Alpine House Archive and Retention Policy"
    ```
4. 您可以重新執行命令，在步驟 2 來確認已變更指派給預設信箱計劃的保留原則。
