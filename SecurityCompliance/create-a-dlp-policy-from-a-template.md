# <a name="create-a-dlp-policy-from-a-template"></a>從範本建立 DLP 原則

要開始使用 DLP 原則，最簡單且最常見方式是使用 Office 365 所包含的其中一個範本。 您可以使用其中一個範本, 或自訂規則, 以符合組織的特定規范需求。
  
Office 365 包含 40 多個現成可用的範本，可協助您符合各種常見法規與商業原則需求。例如，下列項目就有 DLP 原則範本：
  
- Gramm-Leach-Bliley Act (GLBA)
    
- 支付卡行業資料安全標準 (PCI DSS)
    
- 美國個人識別資訊 (美國 PII)
    
- 美國健康保險法案 (HIPAA)
    
您可以修改任何現有的規則或新增規則，以微調範本。例如，您可以將新的敏感資訊類型新增至規則、修改規則中的計數使其更難以或容易觸發、允許人員藉由提供正當業務理由來覆寫規則中的動作，或變更通知和事件報告的傳送對象。DLP 原則範本可做為許多常見規範案例的彈性起始點。
  
您也可以選擇自訂範本 (沒有任何預設規則)，然後從頭設定 DLP 原則，以符合組織的特定規範需求。
  
## <a name="example-identify-sensitive-information-across-all-onedrive-for-business-sites-and-restrict-access-for-people-outside-your-organization"></a>範例: 識別所有商務用 OneDrive 網站上的敏感資訊, 並限制組織外部人員的存取

商務用 OneDrive 帳戶可讓組織中的人員輕鬆共同作業和共用檔。 但對合規性專員的共同問題, 就是儲存在商務用 OneDrive 帳戶中的敏感資訊可能不小心與組織外部的人員共用。 DLP 原則有助於降低此風險。
  
在此範例中, 您將建立一個用來識別美國 PII 資料的 DLP 原則, 其中包括個別的納稅人識別碼 (ITIN)、社會保險號碼和美國護照號碼。 您將會開始使用範本, 然後修改範本以符合組織的符合性需求, 特別是, 您可以:
  
- 新增一種類型的機密資訊 (例如: 銀行帳戶號碼和美國駕駛執照號碼), 如此一來, DLP 原則就能保護更多機密資料。
    
- 使原則更加敏感, 讓敏感資訊的單一發生足以限制外部使用者的存取。
    
- 讓使用者藉由提供正當業務理由或回報誤判來覆寫動作。 如此一來, 您的 DLP 原則將無法防止貴組織中的人員完成工作, 前提是他們有有效的業務理由共用敏感資訊。
    
### <a name="create-a-dlp-policy-from-a-template"></a>從範本建立 DLP 原則

1. 移至 [https://protection.office.com](https://protection.office.com)。
    
2. 使用公司或學校帳戶登入 Office 365。 您現在是在 Office 365 安全性&amp;與合規性中心。
    
3. 在&amp;安全性與合規性\>中心左側導覽\> **資料遺失防護** \> **原則** \> **+ 建立原則**。
    
    ![[建立原則] 按鈕](media/b1e48a08-92e2-47ca-abdc-4341694ddc7c.png)
  
4. 選擇保護您\> **接下來**所需之敏感資訊類型的 DLP 原則範本。
    
    在此範例中, 您將選取**隱私權** \> **美國個人身分識別資訊 (PII) 資料**, 因為它已包含您想要保護的大多數敏感資訊類型, 因此您將在稍後新增一些內容。 
    
    當您選取範本時, 您可以閱讀右邊的說明, 以瞭解範本所保護的敏感資訊類型。
    
    ![用於選擇 DLP 原則範本的頁面](media/775266f6-ad87-4080-8d7c-97f2e7403b30.png)
  
5. 將原則\>命名為 **[下一步]**。
    
6. 若要選擇您想要讓 DLP 原則保護的位置, 請執行下列其中一項操作:
    
  - 選擇 [ \> **下一步]****的 Office 365 中的所有位置**。
    
  - 選擇 [**下一步]**。 **** \> 在這個範例中, 選擇此。
    
    若要包含或排除整個位置 (例如所有 Exchange 電子郵件或所有 OneDrive 帳戶), 請開啟或關閉該位置的**狀態**。 
    
    若要只包含特定的 SharePoint 網站或商務用 OneDrive 帳戶, 請將**狀態**切換為 [開啟], 然後按一下 [**包含**] 下的連結, 選擇 [特定網站] 或 [帳戶]。 當您將原則套用至網站時，該原則中設定的規則會自動套用至該網站的所有子網站。 
    
    ![DLP 原則可以套用的位置選項](media/ee50a61a-e867-4571-a150-3eec8d83650f.png)
  
    在此範例中, 若要保護儲存在所有商務用 OneDrive 帳戶中的機密資訊, 請關閉**Exchange 電子郵件**和**SharePoint 網站**的**狀態**, 並保留**OneDrive 帳戶**的**狀態**。
    
7. 選擇 [**使用高級設定** \> **] [下一步]**。
    
8. DLP 原則範本包含預先定義的規則，且附有條件和動作，分別會偵測特定類型的敏感資訊和採取因應措施。 您可以編輯、刪除或關閉任何現有的規則, 或新增新的規則。 完成後, 按 **[下一步]**。
    
    ![在 US PII 原則範本中擴充的規則](media/3bc9f1b6-f8ad-4334-863a-24448bb87687.png)
  
    在此範例中, 美國 PII 資料範本包含兩個預先定義的規則:
    
  - 偵測**到美國的內容量下限**此規則會尋找包含三種敏感資訊 (ITIN、SSN 和美國護照號碼) 的每一種類型的檔案, 其中的檔案與組織外部人員共用的檔案。 如果找到, 規則會將電子郵件通知傳送給主要網站集合管理員、檔擁有者, 以及上次修改檔的人員。 
    
  - **美國 PII 偵測到大量內容**此規則會尋找包含10個以上的敏感資訊類型的檔案, 以及與組織外部人員共用檔案的檔。 如果找到, 這個動作也會傳送電子郵件通知, 而且它會限制對檔案的存取。 對於商務用 OneDrive 帳戶中的內容, 這表示檔的許可權會受到限制, 但主要網站集合管理員、檔擁有者以及上次修改檔的人員除外。 
    
    若要符合貴組織的特定需求, 您可能會想要更容易觸發規則, 讓敏感資訊的單一發生足以封鎖外部使用者的存取。 在查看這些規則之後, 您會明白您不需要低和高的計數規則--只要發現任何敏感資訊, 就只需要一個規則來封鎖存取。
    
    因此, 您會展開名為「**低」內容量的規則 (偵測到美國 PII** \> **刪除規則**)。
    
    ![刪除規則按鈕](media/bc36f7d2-0fae-4af1-92e8-95ba51077b12.png)
  
9. 現在, 在此範例中, 您必須新增兩個機密資訊類型 (美國銀行帳戶號碼和美國駕駛執照號碼)、允許使用者覆寫規則, 以及將計數變更為任何發生的專案。 您可以編輯一個規則來執行這項操作, 因此請選取 [美國 PII \> **編輯規則**偵測**到大量內容**]。
    
    ![編輯規則按鈕](media/eaf54067-4945-4c98-8dd6-fb2c5d6de075.png)
  
10. 若要新增機密資訊類型, 請在**** [條件\> ] 區段中**新增或變更類型**。 然後, 在 **[新增或變更類型** \> ] 下, 選擇 [**新增** \>選取**u.s. 銀行帳戶號碼**和**美國司機的授權號碼** \> **** \> ** **]。
    
    ![新增或變更類型的選項](media/c6c3ae86-f7db-40a8-a6e4-db11692024be.png)
  
    ![新增或變更類型窗格](media/fdbb96af-b914-4a6c-a97b-bbd014689965.png)
  
11. 若要變更計數 (觸發規則所需的機密資訊實例數目), 請在 [**實例計數** \> ] 下, 選擇每種類型\>的**最小**值輸入1。 最小計數不能為空白。 最大計數可以是空的。空的**最大**值會轉換為**any**。
    
    完成後, 所有敏感資訊類型的最小計數都應該是 1, 而最大計數則應**** 為**1** 。 換句話說, 任何出現的敏感資訊類型都將符合此條件。
    
    ![敏感資訊類型的實例計數](media/5c6e08cb-59a9-4558-b54b-d899836d4737.png)
  
12. 對於最後的自訂, 您不想讓 DLP 原則在使用者有有效的業務理由或遇到假肯定的情況下, 封鎖他們的工作, 因此您希望使用者通知包含覆寫封鎖動作的選項。
    
    在 [**使用者通知**] 區段中, 您可以看到範本中的此規則預設會開啟電子郵件通知和原則提示。 
    
    在 [**使用者覆寫**] 區段中, 您可以看到已開啟商務理由的覆寫, 但不會覆寫報告誤報。 **如果將規則報告為 false 陽性**, 請選擇 [自動覆寫規則]。
    
    ![[使用者通知] 區段和 [使用者覆寫] 區段](media/62720e7a-a939-4c03-b414-67748f3d64a0.png)
  
13. 在規則編輯器的頂端, 將此規則的名稱從預設的**高大量內容**變更為偵測到**美國 pii 的任何內容**, 因為它現在是由其敏感資訊類型的任何出現所觸發。 
    
14. 在規則編輯器\>的底部**儲存**。
    
15. 在\> **下一步**回顧此規則的條件和動作。
    
    請注意規則的**狀態**參數。 如果您關閉整個原則, 則也會關閉包含在該原則中的所有規則。 不過, 您可以在這裡關閉特定規則, 而不會關閉整個原則。 這在您需要調查會產生大量誤判的規則時會很有用。 
    
16. 在下一個頁面上, 閱讀並瞭解下列各項, 然後選擇是否要開啟規則, 或在\> **下一步**進行測試。
    
     建立 DLP 原則之前，您應考慮逐漸推出這些原則，以便在完全強制執行之前評估其影響及測試其效果。 例如, 您不想讓新的 DLP 原則無意間封鎖使用者需要完成工作所需的數千份檔的存取。 
    
    如果您要建立的 DLP 原則有較大的潛在影響, 建議遵循下列順序:
    
17. 以測試模式啟動但不顯示原則提示，然後使用 DLP 報告來評估影響。您可以使用 DLP 報告來檢視原則相符項目的號碼、位置、類型和嚴重性。根據結果，您可以視需要微調規則。在測試模式中，DLP 原則不會影響您的組織中工作人員的生產力。 
    
18. 移至測試模式並顯示通知和原則提示，以便您開始教導使用者相關規範原則及熟悉即將套用的規則。在這個階段，您也可以要求使用者回報誤判，以便您進一步調整規則。
    
19. 開啟原則, 以強制執行規則和內容受保護。 繼續監視 DLP 報告以及任何事件報告或通知，確保得到您想要的結果。 
    
    ![使用測試模式和開啟原則的選項](media/49fafaac-c6cb-41de-99c4-c43c3e380c3a.png)
  
20. 檢查您的設定您的\>此原則選擇 [**建立**]。
    
建立及開啟 DLP 原則之後, 它會部署至其所包含的任何內容來源, 例如 SharePoint Online 網站或商務用 OneDrive 帳戶, 其中原則會自動開始在該內容上強制執行其規則。
  
## <a name="view-the-status-of-a-dlp-policy"></a>檢視 DLP 原則的狀態

您可以隨時在安全性&amp;與合規性中心的 [**資料遺失防護**] 區段中, 于 [**原則**] 頁面上查看 DLP 原則的狀態。 您可以在這裡找到重要資訊，例如是否已順利啟用或停用原則，或原則是否處於測試模式。 
  
下列是不同狀態及其代表的意義。
  
|**狀態**|**說明**|
|:-----|:-----|
|**正在開啟…** <br/> |正在將原則部署到其包含的內容來源。尚未在所有來源上強制執行此原則。  <br/> |
|**測試，有通知** <br/> |原則處於測試模式。不會套用規則中的動作，但會收集原則相符項目，並且可使用 DLP 報告加以檢視。原則相符項目的相關通知會傳送給指定收件者。  <br/> |
|**測試，沒有通知** <br/> |原則處於測試模式。不會套用規則中的動作，但會收集原則相符項目，並且可使用 DLP 報告加以檢視。原則相符項目的相關通知不會傳送給指定收件者。  <br/> |
|**On** <br/> |原則在使用中且已強制執行。原則已成功部署到其所有內容來源。  <br/> |
|**正在關閉…** <br/> |正在將原則從其包含的內容來源移除。此原則在某些來源上仍然在作用中且已強制執行。關閉原則可能需要 45 分鐘。  <br/> |
|**關閉** <br/> |原則不在作用中且並未強制執行。原則的設定 (來源、關鍵字、期限等) 已儲存。  <br/> |
|**刪除 .。。** <br/> |正在刪除原則。原則不在作用中且並未強制執行。  <br/> |
   
## <a name="turn-off-a-dlp-policy"></a>關閉 DLP 原則

您可以隨時編輯或關閉 DLP 原則。 關閉原則會停用原則中的所有規則。
  
若要編輯或關閉 DLP 原則, 請在 [**原則**] \>頁面上選取\> [原則]**編輯原則**。
  
![[編輯原則] 按鈕](media/ce319e92-0519-44fe-9507-45a409eaefe4.png)
